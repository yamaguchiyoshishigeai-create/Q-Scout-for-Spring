# Docs Guide

`docs` 配下の文書は、現行の正本、履歴資料、削除済みの空ファイル整理結果が混在しないように、以下の方針で整理しています。

## 現行正本

- リポジトリ全体の利用案内と起動方法: `../README.md`
- 英語要約版 README: `README.en.md`
- 現行の要件・方針・主要仕様:
  - `20_要件定義/Q-Scout-for-Spring-Web化フェーズ-Web-MVP-最小要件定義書-v1.0.txt`
  - `20_要件定義/Q-Scout-for-Spring-Web化フェーズ-全体設計方針書-v1.0.txt`
  - `20_要件定義/Q-Scout-for-Spring_多言語化対応仕様書_v1.0.txt`
  - `20_要件定義/Web実行制約・運用前提-v0.1.txt`

## 現行補助仕様

- 実装骨格や責務分離の補助説明:
  - `20_要件定義/Q-Scout-for-Spring-Web化フェーズ-既存CLI資産からの移行・流用方針-v1.0.txt`
  - `20_要件定義/Q-Scout-for-Spring-Web化フェーズ-最小Webアーキテクチャ設計書-v1.0.txt`
    - 現行実装の主要骨格を把握するための設計書です。`WebPreviewController` などの後続追加まで完全網羅した正本ではないため、必要に応じてソースコードと併読してください。
- ルール解釈の補助仕様:
  - `20_要件定義/ControllerToRepositoryDirectAccessRule_見直し方針書.txt`
- 現行フロント差分設計:
  - `40_詳細設計/*.md`

## 履歴資料

- 初期の企画書、CLI MVP 前提の要件定義、最小 API 設計は `90_アーカイブ/` に移動しました。
- これらは当時の意思決定や経緯を残すための資料であり、現行実装の正本ではありません。

## 削除済みの空ファイル

以下は中身が空で、正本に見えるだけの状態だったため削除しました。

- `01_プロジェクト計画・管理/プロジェクト計画.txt`
- `30_基本設計/基本設計.txt`
- `50_実装/実装.txt`

- `70_結合テスト/結合テスト.txt`

必要になった場合は、現行実装を基準に新規作成してください。

## フォルダ維持方針

- 正式文書名の空ファイルは置かない
- 空ディレクトリを維持する必要がある場合は、用途説明付きの `README.md` または `.gitkeep` を使う
- 現在は、意味を残したいフォルダに `README.md` を配置して運用意図を明示している
