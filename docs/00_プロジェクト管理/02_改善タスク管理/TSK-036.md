# TSK-036 TSK台帳整合チェック導線と自動化を整備する

- 区分: 運用 / 自動化 / docs
- 件名: TSK台帳整合チェック導線を制度化する
- 状態: 確認待ち
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: `scripts/check_tsk_registry.py`, `scripts/test_tsk_registry.py`, 改善タスク管理ルール, ChatGPTリポジトリ編集運用ルール, 解決済み化専用PR, TSK台帳整合チェッカー利用手順

## 発端

TSK-030 により、改善タスク一覧と個別TSKファイルの整合を静的検査する `scripts/check_tsk_registry.py` と回帰テスト `scripts/test_tsk_registry.py` が追加された。

また、利用手順も追加され、ユーザーのローカル確認で現行台帳PASSと回帰テスト9/9 PASSが確認された。

一方で、TSK-030の作業中に以下の残課題が確認された。

- `改善タスク管理ルール.md` へのチェッカー実行導線追記が未対応である。
- 新規TSK追加後に必ずチェッカーを実行する制度化が未対応である。
- 解決済み化専用PR作成時のチェック導線が未対応である。
- ChatGPT(リポジトリ編集)の台帳更新・PR作成フローへの明文化が未対応である。
- 将来的なCI自動チェックが未対応である。

## スコープ分離方針

本タスクには、手動チェック導線の制度化と、GitHub Actions等による自動起動・CI検証の2種類の作業が混在していた。

Codex利用制限下でCI定義追加・自動実行確認・失敗ログ切り分けまでを同一タスクで扱うと、検証不足のままCloseする危険がある。

そのため、本タスクでは以下に限定して対応する。

- 新規TSK追加後の手動チェック導線の制度化
- TSK状態変更後の手動チェック導線の制度化
- 解決済み化専用PR作成時の手動チェック導線の制度化
- ChatGPT(リポジトリ編集)の台帳更新フローへの明文化
- CI自動実行・自動起動検証は TSK-042 へ分離

## 目的

TSK-030で追加したTSK台帳整合チェッカーを、単体ツールではなく、新規TSK追加・状態変更・解決済み化・台帳更新フローの標準ゲートとして定着させる。

本タスクでは、運用正本への追記、ChatGPT(リポジトリ編集)の作業フローへの制度化、解決済み化専用PRへの適用方針の明文化を対象とする。

CI自動実行の確立は、TSK-042「TSK台帳整合チェッカーのCI自動実行を整備する」で扱う。

## 対応項目

### 1. 改善タスク管理ルールへの追記

`改善タスク管理ルール.md` に、以下を追記する。

- 新規TSK追加後は、原則として `scripts/check_tsk_registry.py` を実行すること。
- TSK状態変更後は、原則として `scripts/check_tsk_registry.py` を実行すること。
- 解決済み化作業後は、原則として `scripts/check_tsk_registry.py` を実行すること。
- チェッカー本体または台帳ルールを変更した場合は `scripts/test_tsk_registry.py` を実行すること。
- `FAIL` が出た場合は、原則としてPR作成またはmerge前に台帳・個別TSK・リンク先を修正すること。
- チェッカーを実行できない場合は、PR本文または作業報告に未実行理由を記録すること。

### 2. 新規TSK追加後チェックの制度化

ChatGPT(リポジトリ編集) が新規TSKを追加した場合、以下を行う運用を明文化する。

- 個別 `TSK-***.md` 追加後に一覧正本へ行を追加する。
- 採番、状態、リンク先、個別ファイル内状態を整合させる。
- PR作成前またはPR本文更新前に `scripts/check_tsk_registry.py` を実行する。
- チェッカーを実行できない場合は、PR本文の `unverified` または作業報告に未実行理由を記録する。

### 3. 解決済み化専用PRへの適用

解決済み化専用PRにおいても、以下をチェック対象とする。

- 通常配置側 `TSK-***.md` が削除されていること。
- `解決済み/TSK-***.md` が追加されていること。
- 一覧正本の状態が `解決済み` へ更新されていること。
- 一覧正本のリンク先が `解決済み/TSK-***.md` へ更新されていること。
- 自律merge前の条件確認に、台帳整合チェックPASSまたは未実行理由記録を含めること。

### 4. ChatGPT(リポジトリ編集)の台帳更新フローへの明文化

`ChatGPTリポジトリ編集運用ルール.md` または関連運用規程へ、以下を追記する。

- 台帳更新後にTSK台帳整合チェッカーを利用する。
- PR本文更新後にも必要に応じて再チェックする。
- 台帳整合チェック結果をPR本文または作業報告に記録する。

### 5. CI自動チェックの分離

GitHub Actions等による自動チェックは、本タスクでは実装しない。

以下は TSK-042 へ分離する。

- `docs/00_プロジェクト管理/02_改善タスク管理/**` が変更されたPRで `scripts/check_tsk_registry.py` を自動実行する仕組み
- `scripts/check_tsk_registry.py` 変更時に `scripts/test_tsk_registry.py` を自動実行する仕組み
- CI上でのPASS/FAILログ確認
- 失敗時ログの切り分け
- GitHub Actions定義の追加または変更

## 完了条件

- TSK-030の未反映事項が本タスクへ移管されている。
- `改善タスク管理ルール.md` へチェッカー実行導線が追記されている。
- 新規TSK追加後・状態変更後・解決済み化後の台帳チェック運用が明文化されている。
- 解決済み化専用PRへの台帳チェック適用方針が整理されている。
- ChatGPT(リポジトリ編集)の台帳更新フローへチェック導線が組み込まれている。
- CI自動実行・自動起動検証が TSK-042 へ分離されている。
- `scripts/check_tsk_registry.py` による台帳整合チェックの実行結果、または未実行理由がPR本文に記録されている。
