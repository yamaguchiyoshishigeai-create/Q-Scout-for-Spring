# TSK-018 Public Free版の完成基準を棚卸しする

- 区分: docs / ポートフォリオ訴求 / 運用
- 件名: Public Free版の完成基準を棚卸しする
- 状態: 未解決
- 優先度: 最優先
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: `README.md`, `docs/00_プロジェクト管理/01_プロジェクト計画/`, Web UI, サンプル評価, CI

## 発端
Public Free版 / Private Pro版の分離運用方針を追加し、Spring Boot 3.5.x 系までを Public Free版の基本対象とした。次に、Public Free版として「無料で試せる範囲はきちんと動く」状態にするための完成基準を明確化する必要がある。

## 現状認識
README、Web UI、サンプル評価、CI、レポート品質、多言語化、ルール詳細ページなど、Public Free版として不足している可能性のある項目が会話中で複数挙がっているが、体系的な完成基準としてはまだ台帳化・文書化されていない。

## 対応方針
Public Free版の完成基準を棚卸しし、以下を整理する。

- Free版に必須の機能
- Free版で不足している説明・導線
- Pro版に回すべき機能
- READMEに記載すべき範囲
- docsに残すべき範囲
- 完成判定に必要な検証項目

必要に応じて、`Public-Free版・Private-Pro版分離運用方針.md` または新規の完成基準文書へ反映する。

## 完了条件
Public Free版の完成基準が一覧化され、必要に応じて docs に正本または補助文書として反映される。以後の改善タスクがこの完成基準に照らして優先度判断できる状態になる。
