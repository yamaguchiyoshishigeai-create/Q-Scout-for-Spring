# TSK-043 改善タスク台帳の正本・生成物分離を設計・実装する

- 区分: 運用 / docs / 自動化 / 課題管理
- 件名: 改善タスク台帳の正本・生成物分離を設計・実装する
- 状態: 確認待ち
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: `改善タスク課題一覧.md`, 個別 `TSK-***.md`, `解決済み/TSK-***.md`, `scripts/check_tsk_registry.py`, `scripts/generate_tsk_registry.py`, `scripts/test_generate_tsk_registry.py`, `TSK台帳整合チェッカー利用手順.md`, `ChatGPTリポジトリ編集運用ルール.md`, `TSK-044.md`

## 発端

`改善タスク課題一覧.md` は、TSK-043まで登録された時点で、実質的にSQLテーブルに近い構造を持つ巨大なMarkdown表になっている。

現行の一覧は、人間が俯瞰しやすい一方で、ChatGPT(リポジトリ編集)が更新する場合はファイル全文に近い置換操作になりやすく、局所修正のつもりでも無関係行へ混入変更が発生するリスクが高まりつつある。

直近のTSK追加・解決済み化作業でも、一覧の一部既存行に軽微な文言混入が発生しかけ、追加修正で戻した。

このため、個別 `TSK-***.md` を正本レコード候補、`改善タスク課題一覧.md` を生成物または索引用ファイルとして扱う方式を設計し、同時に最小実装として生成スクリプトを追加する。

## 目的

改善タスク台帳の更新安全性と一覧性を両立するため、巨大Markdown表を手編集し続ける運用を避け、1タスク1ファイル単位の局所更新を基本にできる状態へ移行する。

本タスクでは、方針整理だけで終わらせず、個別TSKファイル群から `改善タスク課題一覧.md` を生成・比較できる最小スクリプトと回帰テストを追加する。

## 採用方針

採用方針は以下とする。

**個別 `TSK-***.md` / `解決済み/TSK-***.md` を将来の正本候補とし、`改善タスク課題一覧.md` は生成物または索引へ段階移行する。**

ただし、現時点で `改善タスク課題一覧.md` を即時廃止しない。移行期間中は、現行の一覧と個別TSKの双方を整合させる。

## 実装内容

### 1. `scripts/generate_tsk_registry.py` の追加

個別TSKファイル群から、改善タスク課題一覧を生成または比較するスクリプトを追加する。

対応範囲は以下とする。

- 通常配置の `TSK-*.md` を走査する。
- `解決済み/TSK-*.md` を走査する。
- タイトル行から管理IDと件名を抽出する。
- 先頭メタデータから以下を抽出する。
  - 区分
  - 件名
  - 状態
  - 優先度
  - 実施主体
  - 関連箇所
- 既存一覧にある概要、完了条件、根拠 / 関連PR / 備考は、現時点では既存一覧から引き継ぐ。
- 個別ファイルの配置と状態が一致しない場合は失敗させる。
- 生成結果と既存一覧を比較できるようにする。
- 生成結果が既存一覧と異なる場合、TSK単位・列単位の差分診断を出力する。

### 2. 実行モード

`generate_tsk_registry.py` は以下のモードを持つ。

    python scripts/generate_tsk_registry.py --mode summary
    python scripts/generate_tsk_registry.py --mode check
    python scripts/generate_tsk_registry.py --mode generate --output <path>
    python scripts/generate_tsk_registry.py --mode generate --write

各モードの目的は以下である。

- `summary`: 個別TSKから生成対象件数、最大TSK番号、状態別件数を表示する。
- `check`: 個別TSKから生成した一覧と、コミット済み `改善タスク課題一覧.md` を比較する。
- `generate --output`: 生成結果を別ファイルに出力する。
- `generate --write`: 生成結果で `改善タスク課題一覧.md` を上書きする。

### 3. `scripts/test_generate_tsk_registry.py` の追加

生成スクリプトの回帰テストを追加する。

テスト対象は以下とする。

- 生成結果が既存一覧と一致する正常系
- 個別TSKと一覧の差分を検出する異常系
- summaryモードの件数出力
- 個別ファイル配置と状態の不一致検出

### 4. 利用導線の補完

生成チェック導線は以下へ反映する。

- `docs/00_プロジェクト管理/02_改善タスク管理/改善タスク管理ルール.md`
- `docs/00_プロジェクト管理/05_横断運用規程/TSK台帳整合チェッカー利用手順.md`
- `docs/00_プロジェクト管理/05_横断運用規程/ChatGPTリポジトリ編集運用ルール.md`

開始時プロンプトは `scripts/**` を確認対象に含めており、今回の個別コマンド追記対象ではないため、直接更新しない。

### 5. 自動実行未対応項目の分離

GitHub Actions等による `generate_tsk_registry.py` と `test_generate_tsk_registry.py` の自動実行は、本PRでは実装しない。

この未対応項目は、TSK-044「TSK台帳生成チェックのCI自動実行を整備する」として分離登録する。

## 現行チェッカーとの責務分担

### `scripts/check_tsk_registry.py`

引き続き以下を担当する。

- 一覧と個別TSKの採番整合
- 一覧と個別TSKの状態整合
- 通常配置 / 解決済み配置の整合
- リンク先の存在確認
- 最大TSK番号・行数の確認

### `scripts/generate_tsk_registry.py`

新たに以下を担当する。

- 個別TSKから一覧行を生成する。
- 生成結果と既存一覧を比較する。
- 状態別件数・最大TSK番号を個別TSK側から算出する。
- 将来、一覧を生成物または索引へ移行するための足場を提供する。

## 現時点での移行範囲

本タスクでは、以下を実施済みとする。

- 正本候補を個別TSK Markdownへ寄せる方針を明確化した。
- 一覧を生成物または索引へ移行する足場を作った。
- 生成スクリプトを追加した。
- 生成スクリプトの回帰テストを追加した。
- 関連運用規程へ利用導線を補完した。
- CI自動実行未対応分をTSK-044として分離登録した。

本タスクでは、以下は実施しない。

- `改善タスク課題一覧.md` の廃止
- 一覧の番号レンジ別分割
- `check_tsk_registry.py` の全面改修
- GitHub Actionsへの組み込み
