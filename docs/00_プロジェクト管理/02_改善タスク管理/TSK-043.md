# TSK-043 改善タスク台帳の正本・生成物分離を設計する

- 区分: 運用 / docs / 自動化 / 課題管理
- 件名: 改善タスク台帳の正本・生成物分離を設計する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集) / Codex
- 関連箇所: `改善タスク課題一覧.md`, 個別 `TSK-***.md`, `解決済み/TSK-***.md`, `scripts/check_tsk_registry.py`, 将来の `scripts/generate_tsk_registry.py`

## 発端

`改善タスク課題一覧.md` は、TSK-042まで登録された時点で、実質的にSQLテーブルに近い構造を持つ巨大なMarkdown表になっている。

現行の一覧は、人間が俯瞰しやすい一方で、ChatGPT(リポジトリ編集)が更新する場合はファイル全文に近い置換操作になりやすく、局所修正のつもりでも無関係行へ混入変更が発生するリスクが高まりつつある。

直近のTSK追加・解決済み化作業でも、一覧の一部既存行に軽微な文言混入が発生しかけ、追加修正で戻した。

このため、単純な一覧分割だけでなく、個別 `TSK-***.md` を正本レコード、`改善タスク課題一覧.md` を生成物または索引用ファイルとして扱う方式を検討する必要がある。

## 目的

改善タスク台帳の更新安全性と一覧性を両立するため、台帳の正本と生成物の役割分担を設計する。

特に、ChatGPT(リポジトリ編集)が巨大Markdown表を全文更新し続ける運用を避け、1タスク1ファイル単位の局所更新を基本にする。

## 対応方針

### 1. 現行台帳構造の整理

以下の関係を整理する。

- `改善タスク課題一覧.md` が保持している列情報
- 個別 `TSK-***.md` / `解決済み/TSK-***.md` が保持しているメタデータ
- 一覧と個別ファイルで重複している項目
- 一覧にだけ存在する項目
- 個別ファイルに寄せるべき項目

### 2. 正本候補の決定

以下のいずれを正本にするか検討する。

- 個別 `TSK-***.md` を正本とし、一覧を生成物または索引にする案
- YAML / JSON 等の構造化ファイルを正本にし、Markdownを生成する案
- 現行Markdown一覧を正本のまま、番号レンジ別に分割する案

現時点の有力案は、既存資産を活かすため、個別 `TSK-***.md` を正本とし、`改善タスク課題一覧.md` を生成物または索引に寄せる方式である。

### 3. 生成スクリプト方針の検討

将来的に以下のスクリプト追加を検討する。

    scripts/generate_tsk_registry.py

想定機能は以下とする。

- 個別 `TSK-***.md` と `解決済み/TSK-***.md` を走査する
- 先頭メタデータを抽出する
- `改善タスク課題一覧.md` または分割台帳を生成する
- 生成結果と既存一覧の差分を検査できるようにする

### 4. 既存チェッカーとの関係整理

`scripts/check_tsk_registry.py` は現行の一覧正本を前提にしているため、正本・生成物分離時には以下を整理する。

- 現行チェッカーを維持する範囲
- 生成物チェックへ変更する範囲
- 生成スクリプトと整合チェッカーの責務分担
- 分割台帳や生成台帳に対応するための後続タスク要否

### 5. 移行段階の設計

いきなり一覧を生成物化せず、以下の段階移行を検討する。

1. 個別TSKメタデータの記法を固定する
2. 生成スクリプト仕様を決める
3. 既存一覧と生成結果を比較する検証モードを追加する
4. 問題がなければ一覧を生成物または半生成物へ移行する
5. 必要に応じて番号レンジ別分割または索引化を行う

## 完了条件

- `改善タスク課題一覧.md` の肥大化リスクと、ChatGPT(リポジトリ編集)での全文更新リスクが整理されている。
- 個別 `TSK-***.md` を正本にするか、別の構造化ファイルを正本にするかの方針が定まっている。
- `改善タスク課題一覧.md` を生成物、半生成物、または分割台帳として扱う方針が定まっている。
- `scripts/check_tsk_registry.py` との責務分担が整理されている。
- 必要に応じて、生成スクリプト実装やチェッカー改修を後続TSKとして登録できる状態になっている。
- 分割または生成物化を実施する場合の安全な移行手順が整理されている。
