# TSK-023 会話中に発生した課題の改善タスク登録ゲートを運用する

- 区分: 運用 / docs
- 件名: 会話中に発生した課題の改善タスク登録ゲートを運用する
- 状態: 確認待ち
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: 改善タスク管理ルール, 改善タスク課題一覧, チャット運用

## 発端
会話中に有益な修正課題案が複数発生している一方、都度 `改善タスク課題一覧.md` と個別 `TSK-***.md` へ登録されていないため、課題が流れて消える懸念が示された。

## 現状認識
既存の改善タスク管理ルールでは、改善項目は一覧文書と個別タスク文書で管理することになっている。しかし、実際の会話進行中に発生した課題をいつ台帳へ登録するか、どのタイミングで棚卸しするかの運用ゲートは弱い。

## 対応方針
以下の運用を検討・明文化する。

- 会話中に新規課題が発生した場合の一時メモ方法
- 一定の区切りで未登録課題を棚卸しするタイミング
- 台帳登録が必要な課題と不要な会話メモの分別基準
- TSK採番・一覧追記・個別ファイル作成の標準手順
- ChatGPT(リポジトリ編集) と Codex の作業後に未登録課題を確認するゲート

必要に応じて `改善タスク管理ルール.md` へ追記する。

## 実施内容

`改善タスク管理ルール.md` へ、会話中課題の登録ゲートとして以下を追記するPRを作成した。

- 登録対象とする課題
- 登録対象外としてよいもの
- 未登録課題の棚卸しタイミング
- 新規TSK登録手順
- 既存TSKへ統合する場合と新規TSKへ分離する場合の判断基準

## 完了条件
会話中に発生した課題が台帳登録漏れにならないための運用ゲートが明文化され、以後の作業で参照できる状態になる。
