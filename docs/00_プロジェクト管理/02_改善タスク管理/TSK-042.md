# TSK-042 TSK台帳整合チェッカーのCI自動実行を整備する

- 区分: 運用 / 自動化 / CI
- 件名: TSK台帳整合チェッカーのCI自動実行を整備する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → Codex / ユーザー(人)
- 関連箇所: `scripts/check_tsk_registry.py`, `scripts/test_tsk_registry.py`, `.github/workflows/`, 改善タスク管理ルール, TSK台帳整合チェッカー利用手順

## 発端

TSK-036の検討中に、TSK台帳整合チェック導線には、手動運用として制度化できる部分と、GitHub Actions等による自動起動・CI検証が必要な部分が混在していることが確認された。

現状はCodex利用制限が厳しく、CI定義の追加、PR上での自動実行確認、失敗時ログの切り分けまでを同一タスク内で完了させると、検証不足のままCloseする危険がある。

そのため、TSK-036では手動チェック導線の制度化を先行し、CI自動実行の確立は本タスクへ分離する。

## 目的

TSK台帳整合チェッカーを、将来的にGitHub Actions等で自動実行できる状態にする。

台帳更新PR、解決済み化専用PR、チェッカー本体変更PRにおいて、台帳不整合やチェッカー回帰不備を早期に検出できるCI導線を整備する。

## 対応方針

### 1. CI対象範囲の整理

以下の変更時に自動チェックを行うか整理する。

- `docs/00_プロジェクト管理/02_改善タスク管理/**` の変更
- `scripts/check_tsk_registry.py` の変更
- `scripts/test_tsk_registry.py` の変更
- 改善タスク管理ルールまたは関連運用規程の変更

### 2. 実行コマンド案

基本候補は以下とする。

    python scripts/check_tsk_registry.py

チェッカー本体または回帰テストに関係する変更では、以下も対象候補とする。

    python scripts/test_tsk_registry.py

### 3. GitHub Actions化の検討

以下を検討する。

- PR時のみ実行するか、push時にも実行するか
- docs台帳変更時だけに絞るか
- Pythonセットアップ方法
- 失敗時のログが台帳修正に十分か
- 一時的な途中作業ブランチでの不整合をどこまで許容するか

### 4. 検証方法

CI設定を追加する場合は、少なくとも以下を確認する。

- 正常な台帳状態でCIがPASSすること
- `scripts/test_tsk_registry.py` がCI上でPASSすること
- 失敗時のログが原因特定に足りること
- PR本文または作業報告にCI実行結果を記録できること

## 完了条件

- TSK台帳整合チェッカーのCI自動実行対象範囲が整理されている。
- GitHub Actions等で実行するコマンドと発火条件が定義されている。
- CI設定を追加する場合は、PR上でPASS結果が確認されている。
- CI設定を見送る場合は、理由と代替の手動確認導線が明文化されている。
- TSK-036との役割分担が明確になっている。
