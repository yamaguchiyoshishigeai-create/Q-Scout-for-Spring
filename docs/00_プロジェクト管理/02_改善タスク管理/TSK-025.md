# TSK-025 Codex節約型作業分担ルールを整備する

- 区分: 運用 / docs
- 件名: Codex節約型作業分担ルールを整備する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: Codex連携運用ルール, ChatGPTリポジトリ編集運用ルール, 改善タスク管理ルール, CodexExec.result運用

## 発端
Codex の週次使用残量が限られている状況で、Codex が適切と思われる作業であっても、ChatGPT(リポジトリ編集)で実行可能な部分を先に切り出し、Codexには実行・テスト・ログ確認・複数ファイル一括修正など、Codexでなければ効率が悪い作業だけを渡したいという運用要望が出た。

## 現状認識
現行ルールでは、ChatGPT(会話内)、ChatGPT(リポジトリ編集)、Codex、ユーザー(人)の作業主体区分は存在する。一方で、Codex使用量が逼迫している場合に、どの作業をChatGPT側へ寄せ、どの作業だけをCodexへ渡すかの節約型判断基準は明文化されていない。

Codexは、コードベース規模、作業の複雑さ、長時間タスク、保持文脈の大きさにより使用量が増えるため、作業を細分化し、Codexに渡す範囲を最小化する運用規律が必要である。

## 対応方針
以下を整理し、必要に応じて既存運用規程へ反映する。

1. Codex使用量が少ない場合の優先判断基準を定義する。
2. ChatGPT(会話内)で先に行うべき作業を定義する。
3. ChatGPT(リポジトリ編集)で先に切り出すべき作業を定義する。
4. Codexへ渡すべき作業を、実行・テスト・ログ確認・大規模修正・反復修正に限定する。
5. Codex指示書を最小実行単位へ分割する方針を定義する。
6. Codex依頼前に、必ず「ChatGPTで先に処理できる範囲」を検討するゲートを設ける。

## ルール化候補

- docs、README、改善タスク台帳、小規模設定変更は、原則として ChatGPT(リポジトリ編集) を優先する。
- 方針整理、比較検討、作業分解、Codex指示書作成は、原則として ChatGPT(会話内) を優先する。
- build/test 実行、Web起動、CLI実行、ログ切り分けは Codex を優先する。
- 多数ファイルの一括リネーム、リンク総点検、広範囲grep、反復修正は Codex を優先する。
- Codex使用残量が少ない場合、Codex指示書は広範囲一括ではなく、最小検証・最小修正単位に分割する。
- Codexに渡す前に、ChatGPT(リポジトリ編集)で前処理できるファイル確認、台帳更新、README確認、docs確認を先に済ませる。
- Codex実行後は、結果から発生した追加課題を改善タスク台帳へ登録する。

## 完了条件
- Codex節約型の作業分担ルールが文書化される。
- Codex連携運用ルール、ChatGPTリポジトリ編集運用ルール、または改善タスク管理ルールのいずれかに反映される。
- Codex指示書作成時に、ChatGPT側で事前に切り出すべき作業を確認する運用が明確になる。
- TSK-017以降のCodex依頼で、このルールを参照できる状態になる。
