# TSK-020 Private Pro版リポジトリ連携方針を具体化する

- 区分: 運用 / ポートフォリオ訴求 / docs
- 件名: Private Pro版リポジトリ連携方針を具体化する
- 状態: 未解決
- 優先度: 中
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: Public Free版 / Private Pro版分離運用方針, README, 将来のPrivateリポジトリ

## 発端
Public Free版とPrivate Pro版の分離運用方針を追加したが、実際にPrivate Pro版リポジトリを作成・運用する際の連携手順、取り込み方向、漏洩防止、Publicへ戻す修正の基準はまだ概要レベルである。

## 現状認識
現行方針では、Public版をFree試供版兼ポートフォリオ、Private版を商用Pro版の管理先とする整理はできている。一方で、Private版作成時に何をコピーし、何を分離し、どの修正をPublicへ戻すかの運用詳細は未確定である。

## 対応方針
以下を整理する。

- Private Pro版作成時の初期取り込み範囲
- Public → Private の取り込みルール
- Private → Public の還元ルール
- Pro機能の漏洩防止観点
- issue / PR / docs の分離基準
- Public版READMEでどこまでPro版予定機能に触れるか

必要に応じて既存の分離運用方針へ追記する。

## 完了条件
Private Pro版リポジトリを作成する前に必要な連携・分離・還元ルールが文書化され、Public側の運用と矛盾しない状態になる。
