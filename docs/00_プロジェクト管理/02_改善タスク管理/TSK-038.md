## 発端
TSK-029の棚卸しで、`mvnw test`、package、Web起動、CLI起動、サンプル評価などの検証手順を統一する候補が抽出された。

## 目的
検証コマンドの組み立てを都度AI判断に委ねず、標準スクリプトで実行・ログ集約・PASS/FAIL判定できる状態にする。

## 想定対象

- Maven wrapper による test / package 実行
- Web起動確認
- CLI起動確認
- curl 等による主要導線確認
- サンプル評価スクリプトとの接続
- 実行ログのファイル集約
- PASS / FAIL / NOT_RUN の統一出力

## 既存資産

- `scripts/run-sample-evaluation-under-samples.ps1`

この既存スクリプトは、公開サンプル評価の取得、build、Q-Scout実行、結果集約までを一括化しており、本タスクの設計参考とする。

## 実装方針

Windows環境を主前提として、初版は PowerShell を優先する。

必要に応じて sh 版やGitHub Actions化を後続検討する。

- 失敗を修正するか後続TSK化するかの判断
- テスト追加要否
- 検証対象範囲の拡張判断
