# TSK-041 解決済み化PR生成支援ツールを検討する

- 区分: 運用 / 自動化 / PR / 課題管理
- 件名: 解決済み化PR生成支援ツールを検討する
- 状態: 未解決
- 優先度: 中
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集) / Codex
- 関連箇所: 改善タスク課題一覧, 解決済みフォルダ, 解決済み化専用PR, TSK台帳整合チェッカー, scripts

## 発端
TSK-029の棚卸しで、main merge済みの元PRを根拠に、TSK個別ファイルを `解決済み/` へ移動し、一覧正本を更新する作業を支援する候補が抽出された。

TSK-030〜TSK-032が解決済みになり、TSK台帳整合チェッカーも追加されたため、TSK-033の残件登録ゲートにより本候補を後続TSKとして登録する。

## 目的
解決済み化作業における定型差分生成を支援し、状態更新、ファイル移動、一覧リンク更新、根拠記録の漏れを低減する。

## 想定支援項目

- 対象TSK番号の受け取り
- 元PR番号と `merged=true` の確認結果記録支援
- 個別ファイルの `状態: 解決済み` 変更案生成
- `解決済み/TSK-***.md` パス生成
- 通常配置側 `TSK-***.md` 削除案生成
- 一覧正本の該当行更新候補生成
- TSK台帳整合チェッカー実行導線との接続
- 解決済み化専用PR本文テンプレートとの接続

## 実装方針

初期段階では、GitHubへの直接書き込みを行う完全自動化ではなく、Pythonによる差分案生成またはチェックリスト生成に留める。

GitHub操作を伴う完全自動化は、権限・安全性・誤更新リスクを整理したうえで後続検討とする。

## AI判断を残す範囲

- 解決済み化してよいかの最終判断
- 元PRがタスク本義を満たしたかの判断
- 解決済み化専用PRの自律merge可否判断
- 移管すべき残課題の有無判断

## 完了条件

- 解決済み化PR生成支援の安全な適用範囲が整理される
- 完全自動化せず差分案生成に留めるかどうかが判断される
- 必要に応じてチェックリストまたは生成支援スクリプトが追加される
- TSK台帳整合チェッカーとの接続方針が整理される
