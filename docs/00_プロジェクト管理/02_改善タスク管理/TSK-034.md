## 発端
TSK-031 により、CodexプロンプトのGit操作安全性を静的検査する `scripts/check_codex_prompt_git_safety.py` と回帰テスト `scripts/test_codex_prompt_git_safety.py` が追加された。

また、`Codex連携運用ルール.md` には、Codex投入前にチェッカーを実行する一次導線が追加された。

一方で、TSK-031 の作業中に以下の残課題が確認された。

- `CodexPrompt_GitScriptEscalation_Rules.md` への最終チェックリスト追記が未反映である。
- Codexプロンプト作成時の自動実行は未対応である。
- PR作成時のCI自動チェックは未対応である。
- `CodexExec.md` の標準配置・標準ファイル名・テンプレート方針は未整備である。
- ChatGPTがプロンプト提示前に必ずチェックする制度化は一部運用上の約束に留まる。
- README、docs索引、開始時プロンプトからの利用導線がまだ弱い。

## 目的
TSK-031で追加したチェッカーを、単体ツールではなくCodexプロンプト作成フローの標準ゲートとして定着させる。

あわせて、導線の完全化、標準配置、開始時プロンプト連携、README/docs導線、将来的なCI化を整理・実装する。

## 対応項目

### 1. CodexPrompt_GitScriptEscalation_Rules.md への最小追記

`CodexPrompt_GitScriptEscalation_Rules.md` の最終チェックリストまたは関連節へ、以下を最小差分で追記する。

- Codex向けプロンプトをファイル化した場合、Codex投入前に `scripts/check_codex_prompt_git_safety.py` を実行すること。
- スクリプト雛形を含む場合は `--allow-script-blocks` を利用できること。
- チェッカー本体または検査ルールを変更した場合は `scripts/test_codex_prompt_git_safety.py` を実行すること。

既存本文にはGit操作例が多く含まれるため、全文置換ではなく対象節への最小差分追記を優先する。

### 2. CodexExec.md 標準配置・テンプレート方針

Codex向けプロンプトをファイル化する場合の標準を定義する。

検討候補:

- 標準ファイル名: `CodexExec.md`
- 実行結果ファイル名: `CodexExec.result`
- 配置場所: リポジトリルート、または `docs/00_プロジェクト管理/` 配下の一時作業領域
- Git管理対象にするか、ローカル一時ファイルとして扱うか
- テンプレートをdocsに置くか

### 3. ChatGPT提示前チェックの制度化

ChatGPTがCodexプロンプトを提示する前に、以下を案内または実施できる運用を明文化する。

- プロンプトをファイル化する場合はチェッカー実行を案内する。
- チェッカー実行結果が `PASS` であることを確認してからCodexへ投入する。
- `FAIL` の場合は、指摘行を修正して再実行する。
- チェッカーを実行できない場合は、未実行理由を明記する。

### 4. README / docs索引 / 開始時プロンプト導線

以下の導線を整備する。

- READMEまたはdocs索引から、CodexプロンプトGit操作準拠チェッカー利用手順へ到達できるようにする。
- `Q-Scout-for-Spring_開始時プロンプト_完全版.md` または関連開始時導線へ、Codexプロンプト作成時のチェッカー確認を追記する。
- Codex連携運用ルールから利用手順へのリンクを必要に応じて補強する。

### 5. CI自動チェックの検討または実装

GitHub Actions等で、以下のような自動チェックを実施できるか検討する。

- `CodexExec.md` がPRに含まれる場合にチェッカーを実行する。
- `docs/**Codex*.md` または Codexプロンプト相当ファイルが変更された場合にチェッカーを実行する。
- `scripts/check_codex_prompt_git_safety.py` 変更時に `scripts/test_codex_prompt_git_safety.py` を実行する。

初回は検討文書化まででも可とし、CI実装は安全性・対象ファイル誤検知・運用負荷を確認してから判断する。
