# TSK-029 AI運用作業の定式化・スクリプト化候補を棚卸しする

- 区分: 運用 / docs / 自動化
- 件名: AI運用作業の定式化・スクリプト化候補を棚卸しする
- 状態: 確認待ち
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: `scripts/`, 改善タスク管理ルール, ChatGPTリポジトリ編集運用ルール, Codex連携運用ルール, AIリポジトリ作業証跡管理ルール, docs運用全般

## 発端
現状、このリポジトリでは改善タスク管理、PR証跡、解決済み化、Codexプロンプト作成、docs整理など、運用作業の多くをAIの判断と操作に依存している。

一方で、仕様上の手順が明確であり、判断・実行・操作の一部を `ps1` / `sh` 等のスクリプト、または Java / Python 等の通常プログラムとして実行できる領域が存在する。

AI判断に任せる必要がない定型作業をプログラム化すれば、運用ブレ、見落とし、プロンプト作成ミス、Codexトークン消費、ユーザー確認負荷を低減できる可能性がある。

## 現状認識
運用規程や台帳ルールは整備されつつあるが、以下のような作業はまだAIが都度読み取り、判断し、GitHub操作する形になっている。

- 改善タスク一覧と個別TSKの整合確認
- `確認待ち` から `解決済み/` への移動判断と台帳更新
- TSK採番と重複確認
- PR本文の `AI_REPO_RESULT` 証跡確認
- CodexプロンプトのGit操作ルール準拠確認
- docsリンク切れ確認
- docs配置規程違反の確認
- README / docs索引の参照漏れ確認
- i18nキーとテンプレート参照の整合確認
- サンプル評価、Web起動、CLI起動などの検証補助

これらの一部は、AIの裁量判断ではなく、機械的な検査・生成・更新支援へ置き換えられる可能性が高い。

## 対応方針
まずは実装に入らず、現行リポジトリを精査し、AI判断・AI操作に依存している運用作業を棚卸しする。

棚卸しでは、各候補について以下を整理する。

- 対象作業
- 現在の実施方法
- プログラム化できる範囲
- AI判断を残すべき範囲
- 想定実装方式（ps1 / sh / Python / Java / GitHub Actions 等）
- 実装難度
- 優先度
- Codex要否
- 期待効果
- 後続TSK化の要否

## 実施内容

- `docs/00_プロジェクト管理/05_横断運用規程/AI運用作業スクリプト化候補棚卸し.md` を追加。
- 現行のAI依存運用を棚卸しし、プログラム化候補を8件に分類。
- 最初に実装すべき候補として以下3件を推奨。
  1. TSK台帳整合チェッカー
  2. CodexプロンプトGit操作準拠チェッカー
  3. AI_REPO_RESULT証跡チェッカー
- 後続TSK分割案として TSK-030 から TSK-036 の候補を整理。
- 既存スクリプト `scripts/run-sample-evaluation-under-samples.ps1` を、検証コマンド支援スクリプト候補の既存資産として位置づけ。

## 初期候補

### 候補1: TSK台帳整合チェッカー

改善タスク一覧、個別TSK、`解決済み/` 配置、状態、リンク先、重複採番を検査する。

期待効果が大きく、AI判断のブレを減らしやすいため、最初の実装候補として有力。

### 候補2: CodexプロンプトGit操作準拠チェッカー

Codexプロンプト内に、通常コマンドで直接実行すべきでない `git checkout`、`git switch`、`git add`、`git commit`、`git push`、`gh pr create` 等が含まれていないかを検査する。

今回発生した `CodexPrompt_GitScriptEscalation_Rules.md` 未適用問題の再発防止に直結するため、優先度が高い。

### 候補3: AI_REPO_RESULT証跡チェッカー

PR本文または作業結果内に `AI_REPO_RESULT_BEGIN` / `AI_REPO_RESULT_END`、`TASK_ID`、`STATUS`、`changed files`、`verification`、`commit`、`pr` 等が含まれているかを検査する。

### 候補4: docsリンク・配置規程チェッカー

Markdownリンク切れ、docs配置規程違反、索引から辿れない文書を検査する。

### 候補5: i18nキー整合チェッカー

`messages*.properties` と Thymeleaf テンプレート等で参照されるキーの整合を確認する。

### 候補6: 検証コマンド支援スクリプト

Web起動、CLI起動、Maven test/package、サンプル評価などの実行手順を統一し、AIやCodexが手順を都度組み立てる必要を減らす。
