# 改善タスク管理台帳

## 1. 文書目的
本台帳は、Q-Scout for Spring における改善項目を一元管理し、未解決 / 解決中 / 解決済み / 保留の状態を可視化することを目的とする。

本台帳は、バグ票の代替ではなく、主として以下を対象とする。

- 外部評価を踏まえた改善
- README / UI / UX / 訴求文の改善
- docs 運用改善
- 実装方針に基づく継続改善
- 品質向上のための整理済み改善タスク

## 2. 運用ルール
- 改善項目は、原則として本台帳へ集約する
- 各項目には一意の管理IDを付与する
- 状態は少なくとも以下のいずれかで管理する
  - 未解決
  - 解決中
  - 解決済み
  - 保留
- 解決済みにした項目は、必要に応じて根拠ファイルや PR 番号を記録する
- 大規模な実装作業に分解する場合は、関連文書や Pull Request へリンクを張る
- 本台帳は履歴を残すため、項目削除ではなく状態更新を原則とする

## 3. ステータス定義
### 3.1 未解決
課題認識済みだが、未着手の状態。

### 3.2 解決中
対応方針が定まり、作業着手済みの状態。

### 3.3 解決済み
リポジトリ反映、確認、または方針確定まで完了した状態。

### 3.4 保留
課題認識はあるが、優先度・前提条件・依存関係等により当面着手しない状態。

## 5. 改善項目詳細

### TSK-001
- 区分: UI / UX
- 件名: 60秒実行制限の意味を明確化する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集) / Codex
- 関連箇所: トップページ、help、エラー文言
- 発端: 修正点メモ
- 現状認識: 実行制限 60 秒の表示はあるが、タイムアウトの意味や超過時挙動の説明が薄い。
- 対応方針: 文言追加、help ページ追記、必要ならエラー表示の改善を行う。
- 完了条件: 利用者が「何が 60 秒で制限されるか」をトップまたは help で理解できる。
- 関連ファイル: `src/main/resources/templates/index.html`
- 関連PR: なし
- 備考: UI 文言改善として小規模修正から着手しやすい。

- 対応方針: 生成ロジックと出力例を点検し、必要ならルール別出し分けを強化する。
- 完了条件: 指摘内容と改善ヒントの対応が妥当で、画一的に見えない。
- 関連ファイル: レポート生成系 Java クラス、`messages_*.properties`

## 6. 優先度基準
### 6.1 最優先
対外評価、主要導線、利用継続性、重大な誤解、または実利用に強く影響する改善。

### 6.2 高
見せ方、説明力、保守性、主要画面品質に大きく効く改善。

### 6.3 中
価値はあるが、直近の主要評価や主要導線には直結しない改善。

### 6.4 低
将来拡張や長期的整理として有効だが、直近優先度は低い改善。
