# TSK-032 AI_REPO_RESULT証跡チェッカーを実装する

- 区分: 運用 / 自動化 / 証跡管理
- 件名: AI_REPO_RESULT証跡チェッカーを実装する
- 状態: 解決済み
- 優先度: 高
- 実施主体: ChatGPT(リポジトリ編集)
- 関連箇所: AIリポジトリ作業証跡管理ルール, PR本文, コミットメッセージ本文, `scripts/`, AI運用作業スクリプト化候補棚卸し

## 発端
TSK-029 の棚卸しにより、AIリポジトリ作業証跡管理ルールで要求される `AI_REPO_RESULT` ブロックや作業証跡項目を、AIが都度手作業で組み立てるだけでは記載漏れが起きる可能性があると整理された。

PR本文、PRコメント、コミットメッセージ本文に残す証跡は、後続確認や履歴追跡の基盤であり、機械的な項目検査に向いている。

## 目的
PR本文または作業結果テキストに、AIリポジトリ作業証跡管理ルールで求められる証跡項目が含まれているかを検査するチェッカーを実装する。

## 想定検査項目

- `AI_REPO_RESULT_BEGIN`
- `AI_REPO_RESULT_END`
- `ACTOR`
- `TASK_ID`
- `TITLE`
- `STATUS`
- `changed files`
- `summary`
- `verification`
- `unverified`
- `commit`
- `push`
- `pr`
- `pr url`

必要に応じて、`ChatGPT-Repo-Result` や `Codex-Result` の短縮記録も検査対象に含める。

## 実装方針

初版は Python CLI として実装した。

想定コマンド例:

    python scripts/check_ai_repo_result.py pr_body.md

短縮証跡だけを明示的に検査する場合は以下を使う。

    python scripts/check_ai_repo_result.py commit_message.txt --format short

`unverified` 欠落を意図的に許容する場合は以下を使う。

    python scripts/check_ai_repo_result.py pr_body.md --allow-missing-unverified

チェッカー本体の回帰テストは以下で実行する。

    python scripts/test_ai_repo_result.py

出力は `PASS` / `FAIL` とし、不足項目がある場合は項目名と推奨追記内容を表示する。

- `scripts/test_ai_repo_result.py` を追加し、正常系・不足系・短縮形式・オプション指定の回帰テストを追加。

## 回帰テスト観点

- 正常なフル証跡ブロックが `PASS` になること。
- フル証跡ブロックの必須項目不足が `FAIL` になること。
- `unverified` 欠落を通常時は `FAIL` にし、オプション指定時は許容できること。
- `ChatGPT-Repo-Result` の短縮証跡が `PASS` になること。
- `Codex-Result` の短縮証跡が `PASS` になること。
- 短縮証跡の必須項目不足が `FAIL` になること。
- 証跡ブロックがないファイルが `FAIL` になること。

## AI判断を残す範囲

- summary の内容妥当性
- verification の意味的十分性
- unverified の網羅性
- 実際の変更内容と証跡内容の整合判断
- 証跡に長大ログや不要情報が混入していないかの判断
