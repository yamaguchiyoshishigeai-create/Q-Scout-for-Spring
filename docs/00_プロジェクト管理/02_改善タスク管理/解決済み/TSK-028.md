# TSK-028 解決済み化PRのChatGPT自律merge運用を整備する

- 区分: 運用 / docs
- 件名: 解決済み化PRのChatGPT自律merge運用を整備する
- 状態: 解決済み
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: 改善タスク管理ルール, ChatGPTリポジトリ編集運用ルール, 解決済み化専用PR

## 発端
TSK-018 および TSK-026 の解決済み化では、ChatGPT(リポジトリ編集) が解決済み化専用PRを作成し、その後ユーザーが main へ merge する流れになっていた。

しかし、解決済み化専用PRは、既にユーザー確認済みまたは main merge 済みの元PRを根拠として、台帳・個別TSK・`解決済み/` フォルダを整合させるだけの後続処理である。

そのため、毎回ユーザーに merge 操作を依頼するより、条件を限定して ChatGPT(リポジトリ編集) が解決済み化専用PRを自律的に merge できる運用にした方が、作業効率と台帳整合性を高められる可能性があった。

## 現状認識
現行ルールでは、解決済み化専用PRの扱いは明文化されているが、そのPRを ChatGPT(リポジトリ編集) が merge してよい条件は未定義であった。

一方で、通常の機能追加、実装修正、設計方針変更、運用規程変更、大きなdocs変更については、ユーザー確認を必須とすべきであり、ChatGPT が自律的に merge する対象へ含めるべきではない。

したがって、対象を「解決済み化専用PR」に限定し、変更内容・根拠・禁止事項・完了報告を明文化した。

## 対応方針
以下を運用規程へ追加した。

1. ChatGPT(リポジトリ編集) が自律 merge してよいPR種別を限定する。
2. 対象を、既に main merge 済みまたはユーザー確認済みの元PRを根拠にした解決済み化専用PRへ限定する。
3. 変更対象を `解決済み/TSK-***.md` 追加、通常配置側 `TSK-***.md` 削除、`改善タスク課題一覧.md` の状態・根拠・リンク更新に限定する。
4. 実装コード、設定、README本文、設計本文、運用規程本文を変更するPRは対象外とする。
5. ChatGPT が merge した場合は、merge後にPR状態を再確認し、完了報告へ記録する。

## 実施内容

- `改善タスク管理ルール.md` に、解決済み化専用PRのChatGPT自律merge条件を追加。
- `ChatGPTリポジトリ編集運用ルール.md` に、ChatGPTが例外的にmerge可能なPR種別と禁止対象を追加。
- `改善タスク課題一覧.md` に TSK-028 を追加。
- Codex により、`ChatGPTリポジトリ編集運用ルール.md` へ最小参照追記を追加。
- ユーザー確認により PR #54 が main へ merge 済み。
- PR #54 main merge 確認後、本ファイルを解決済み化対象として整理。

## 完了条件

- 解決済み化専用PRを ChatGPT が自律 merge してよい条件が明文化される。
- 通常PR、実装PR、設計・運用方針変更PRは対象外であることが明文化される。
- merge後の再確認と完了報告が必須化される。
- 以後、条件を満たす解決済み化専用PRでは、ユーザーの追加merge操作依頼を省略できる。
