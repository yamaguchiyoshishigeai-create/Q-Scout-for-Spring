# TSK-027 GitHub Issues / Projects 正本化を検討する

- 区分: 運用 / docs / 課題管理
- 件名: GitHub Issues / Projects 正本化を検討する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集)
- 関連箇所: GitHub Issues, GitHub Projects, Issue Forms, 改善タスク管理ルール, 改善タスク課題一覧, `TSK-***.md`

## 発端
現行の改善タスク管理は、`TSK-***.md` と `改善タスク課題一覧.md` を正本として運用している。しかし、PR merge 後の状態更新、`確認待ち` から `解決済み` への移動、一覧リンク更新などが手作業になっており、運用負荷と更新漏れが発生している。

GitHub Issues / Projects / Issue Forms を利用すれば、現行 `TSK-***.md` に記載している発端・現状認識・対応方針・完了条件・関連ファイル・関連PR・備考などを Issue 本文へ保持し、状態・優先度・区分・担当などを Projects 側で管理できる可能性がある。

## 現状認識
GitHub Issues はタスク、バグ、アイデア、フィードバック等を追跡するための機能であり、Projects は Issue や Pull Request を計画・追跡するための機能である。Issue Forms を使うことで、入力項目、必須チェック、ラベル、Projects 追加などを YAML で定義でき、入力内容は Issue 本文の Markdown に変換される。

また、Projects には built-in workflow があり、Issue / Pull Request が close された場合や Pull Request が merge された場合に、Project 上の Status を Done にする自動化が利用できる。

一方で、Issue fields は public preview であり、Projects との連携や利用条件に制約があるため、現時点では依存しない方が安全である。

## 対応方針
以下を検討する。

1. 現行 `TSK-***.md` の項目を GitHub Issue / Project へどう写像するか整理する。
2. Issue Form 案を作成する。
3. Project field 案を作成する。
4. Label 案を作成する。
5. 旧 TSK ID を Issue に残す方式を整理する。
6. TSK-md と GitHub Issues / Projects の二重管理期間をどう扱うか整理する。
7. TSK-023 / TSK-025 / TSK-026 などの運用系タスクを試験移行候補として評価する。
8. 無料機能のみで実現可能か、有料機能や preview 機能への依存がないか確認する。
9. Codex 使用量を抑えるため、実装ではなく設計・検討・対応表作成を先行する。

## 検討観点

### 情報保持

- `TSK-***.md` と同等の発端、現状認識、対応方針、完了条件、関連ファイル、関連PR、備考を Issue 本文で保持できるか。
- 状態変更や方針変更の履歴を Issue コメントまたはタイムラインで十分に追跡できるか。
- 旧 TSK 番号を検索可能な形で残せるか。

### 一覧性

- Projects の table / board view で `改善タスク課題一覧.md` と同等以上の一覧性を確保できるか。
- Status、Priority、Category、Actor、Related PR、Legacy TSK ID を適切に管理できるか。

### 自動化

- PR merge 時に Done 化できるか。
- Issue close 時に Done 化できるか。
- Issue Form から Project へ自動追加できるか。
- GitHub Actions 等の独自自動化なしで運用できる範囲はどこまでか。

### 費用・制約

- GitHub Free の範囲で運用できるか。
- Issue fields など preview / private project 限定機能に依存しないか。
- Public Free 版と将来の Private Pro 版で同じ運用を使えるか。

### 移行負荷

- 既存 TSK-001〜TSK-026 をすべて移行する必要があるか。
- 解決済みタスクはアーカイブ扱いで残すか。
- 新規タスクから Issue 起票へ切り替える段階移行が可能か。

- TSK-md管理を廃止できるか、併用すべきか、段階移行すべきかの判断材料が揃う。
