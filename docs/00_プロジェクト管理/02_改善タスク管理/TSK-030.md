# TSK-030 TSK台帳整合チェッカーを実装する

- 区分: 運用 / 自動化 / docs
- 件名: TSK台帳整合チェッカーを実装する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(リポジトリ編集) → Codex
- 関連箇所: `docs/00_プロジェクト管理/02_改善タスク管理/`, `scripts/`, AI運用作業スクリプト化候補棚卸し

## 発端
TSK-029 の棚卸しにより、改善タスク一覧、個別TSK、`解決済み/` 配置、状態、リンク先、採番の整合確認は、AI判断に依存させるより機械検査へ寄せるべき最優先候補として整理された。

TSK-029の登録時には、本義である棚卸し作業の未着手にもかかわらず状態が `確認待ち` となる誤りが発生しており、台帳状態の整合検査を自動化する必要性が実例として確認された。

## 目的
改善タスク管理台帳の整合を機械的に検査できるチェッカーを実装し、AIによる状態誤判定、リンク更新漏れ、配置ミス、採番ミスを低減する。

## 想定検査項目

- `改善タスク課題一覧.md` に記載されたTSK行を抽出できること
- 一覧上のTSK番号と個別 `TSK-***.md` の存在が一致すること
- `解決済み` 状態のタスクが `解決済み/` 配下を参照していること
- `未解決`、`解決中`、`確認待ち`、`保留` のタスクが通常配置側を参照していること
- 個別ファイル内の `状態` と一覧上の `状態` が一致すること
- TSK番号の重複がないこと
- 一覧リンク先が存在すること
- 最大TSK番号を表示できること

## 実装方針

初版は Python CLI を推奨する。

想定コマンド例:

    python scripts/check_tsk_registry.py

出力は `PASS` / `FAIL` とし、失敗時は不整合箇所、対象TSK番号、対象ファイル、期待値、実際値を表示する。

## AI判断を残す範囲

- 不整合が意図的な一時状態かどうかの判断
- 登録PRと実作業PRの扱い
- タスク粒度の再設計
- 解決済み化可否の最終判断

## 完了条件

- TSK台帳整合チェッカーが追加される
- READMEまたは該当docsに実行方法が記載される
- 少なくとも現行台帳に対して実行できる
- PASS / FAIL が明確に分かる
- 不整合時に修正対象が分かる出力になる
