# TSK-030 TSK台帳整合チェッカーを実装する

- 区分: 運用 / 自動化 / docs
- 件名: TSK台帳整合チェッカーを実装する
- 状態: 確認待ち
- 優先度: 高
- 実施主体: ChatGPT(リポジトリ編集)
- 関連箇所: `docs/00_プロジェクト管理/02_改善タスク管理/`, `scripts/`, AI運用作業スクリプト化候補棚卸し

## 発端
TSK-029 の棚卸しにより、改善タスク一覧、個別TSK、`解決済み/` 配置、状態、リンク先、採番の整合確認は、AI判断に依存させるより機械検査へ寄せるべき最優先候補として整理された。

TSK-029の登録時には、本義である棚卸し作業の未着手にもかかわらず状態が `確認待ち` となる誤りが発生しており、台帳状態の整合検査を自動化する必要性が実例として確認された。

## 目的
改善タスク管理台帳の整合を機械的に検査できるチェッカーを実装し、AIによる状態誤判定、リンク更新漏れ、配置ミス、採番ミスを低減する。

## 想定検査項目

- `改善タスク課題一覧.md` に記載されたTSK行を抽出できること
- 一覧上のTSK番号と個別 `TSK-***.md` の存在が一致すること
- `解決済み` 状態のタスクが `解決済み/` 配下を参照していること
- `未解決`、`解決中`、`確認待ち`、`保留` のタスクが通常配置側を参照していること
- 個別ファイル内の `状態` と一覧上の `状態` が一致すること
- TSK番号の重複がないこと
- 一覧リンク先が存在すること
- 最大TSK番号を表示できること

## 実装方針

初版は Python CLI として実装した。

想定コマンド例:

    python scripts/check_tsk_registry.py

チェッカー本体の回帰テストは以下で実行する。

    python scripts/test_tsk_registry.py

出力は `PASS` / `FAIL` とし、失敗時は不整合箇所、対象TSK番号、対象ファイル、期待値、実際値を表示する。

## 実施内容

- `scripts/check_tsk_registry.py` を追加。
- 改善タスク一覧からTSK行を抽出する処理を追加。
- TSK番号重複、リンクラベル不一致、リンク先存在確認を追加。
- `解決済み` と通常配置側のリンク先整合チェックを追加。
- 個別TSKファイル内の `- 状態: ...` と一覧上の状態一致チェックを追加。
- 一覧に載っていない通常配置側 `TSK-*.md` と `解決済み/TSK-*.md` の検出を追加。
- 最大TSK番号の表示を追加。
- `scripts/test_tsk_registry.py` を追加し、正常系・重複・リンク欠落・配置不整合・状態不一致・未掲載ファイル・リンクラベル不一致の回帰テストを追加。
- `docs/00_プロジェクト管理/05_横断運用規程/TSK台帳整合チェッカー利用手順.md` を追加。
- 既存台帳不整合として検出されたTSK-014行の列数不足を修正。
- TSK-030の未反映事項と台帳整合チェック導線強化・自動化課題を、後続タスク TSK-036 へ移管。

## 回帰テスト観点

- 正常な通常タスクと解決済みタスクが `PASS` になること。
- TSK番号重複を検出できること。
- リンク先欠落を検出できること。
- 通常タスクが `解決済み/` 配下へリンクしている不整合を検出できること。
- 解決済みタスクが通常配置側へリンクしている不整合を検出できること。
- 一覧と個別ファイルの状態不一致を検出できること。
- 一覧に載っていない通常配置側ファイルを検出できること。
- 一覧に載っていない解決済みファイルを検出できること。
- リンクラベル不一致を検出できること。
