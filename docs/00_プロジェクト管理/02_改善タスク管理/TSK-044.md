# TSK-044 TSK台帳生成チェックのCI自動実行を整備する

- 区分: 運用 / 自動化 / CI
- 件名: TSK台帳生成チェックのCI自動実行を整備する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → Codex / ユーザー(人)
- 関連箇所: `scripts/generate_tsk_registry.py`, `scripts/test_generate_tsk_registry.py`, `scripts/check_tsk_registry.py`, GitHub Actions, 改善タスク管理ルール, TSK台帳整合チェッカー利用手順

## 発端

TSK-043により、個別 `TSK-***.md` / `解決済み/TSK-***.md` から `改善タスク課題一覧.md` を生成・比較する `scripts/generate_tsk_registry.py` と、回帰テスト `scripts/test_generate_tsk_registry.py` が追加された。

これにより、ChatGPT(リポジトリ編集)やユーザー(人)が手動で台帳生成チェックを実行できるようになった。

一方で、PR作成時やmain merge前に、GitHub Actions等で自動的に以下を実行する仕組みは未整備である。

- `python scripts/generate_tsk_registry.py --mode check`
- `python scripts/test_generate_tsk_registry.py`
- 必要に応じた `python scripts/check_tsk_registry.py`

このため、プログラム自動実行に必要な未対応項目として、本タスクをTSK-043から分離する。

## 目的

TSK台帳生成チェックを手動運用だけに依存せず、GitHub Actions等のCIで自動実行できるようにする。

これにより、個別TSK・一覧台帳・生成スクリプトの不整合をPR段階で検出し、巨大Markdown表の手編集リスクをさらに下げる。

## 対応方針

以下を検討・整備する。

1. GitHub Actions上で `scripts/generate_tsk_registry.py --mode check` を実行する。
2. `scripts/generate_tsk_registry.py` または `scripts/test_generate_tsk_registry.py` 変更時に、生成スクリプト回帰テストを実行する。
3. 改善タスク台帳、個別TSK、解決済みTSK、生成スクリプト、既存台帳チェッカーに関係する変更をトリガー対象にする。
4. `scripts/check_tsk_registry.py` のCI自動実行を扱うTSK-042との役割分担を整理する。
5. PASS/FAILログ確認方法を文書化する。

## TSK-042との関係

TSK-042は、TSK台帳整合チェッカー `scripts/check_tsk_registry.py` のCI自動実行を扱う。

本タスクは、TSK-043で追加された生成チェック `scripts/generate_tsk_registry.py` とその回帰テストのCI自動実行を扱う。

両者は密接に関連するが、対象スクリプトと検査責務が異なるため、別タスクとして管理する。

## 完了条件

- `scripts/generate_tsk_registry.py --mode check` のCI自動実行方針が整理されている。
- `scripts/test_generate_tsk_registry.py` のCI自動実行方針が整理されている。
- GitHub Actions等で実行する場合の対象パス、発火条件、実行コマンドが定義されている。
- TSK-042との役割分担が明確になっている。
- CI実装を行う場合は、PR上でPASS/FAILログを確認できる。
- CI実装を見送る場合は、見送り理由と代替確認手順が文書化されている。
