# TSK-039 i18nキー整合チェッカーを実装する

- 区分: 運用 / 自動化 / i18n
- 件名: i18nキー整合チェッカーを実装する
- 状態: 未解決
- 優先度: 中
- 実施主体: ChatGPT(会話内) → Codex
- 関連箇所: messages.properties, messages_en.properties, Thymeleafテンプレート, TSK-008, scripts

## 発端
TSK-029の棚卸しで、`messages*.properties` と Thymeleaf テンプレート等で参照されるメッセージキーの整合確認候補が抽出された。

TSK-030〜TSK-032が解決済みになったため、TSK-033の残件登録ゲートにより本候補を後続TSKとして登録する。

## 目的
多言語UI文言のキー欠落、未定義参照、言語間キー差分を機械的に検査し、TSK-008関連の文言整合確認を支援する。

## 想定検査項目

- `messages.properties` と `messages_en.properties` 等のキー差分
- Thymeleafテンプレート内の `#{...}` 参照抽出
- 未定義キー検出
- 未使用キー候補検出
- 言語ごとのキー集合差分表示

## 実装方針

初版は Python または Java を候補とする。

軽量検査を優先する場合は Python、Spring/Thymeleafの文脈に寄せる場合は Java を検討する。

## AI判断を残す範囲

- 翻訳品質
- 用語統一
- UI上の自然さ
- 未使用キー候補を本当に削除してよいかの判断

## 完了条件

- i18nキー整合チェッカーの方針が整理される
- messages系ファイル間のキー差分を検出できる
- テンプレート参照キーの未定義を検出できる
- 利用手順または実装指示が記録される
