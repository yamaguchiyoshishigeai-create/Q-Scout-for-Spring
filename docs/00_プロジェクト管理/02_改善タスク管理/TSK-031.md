# TSK-031 CodexプロンプトGit操作準拠チェッカーを実装する

- 区分: 運用 / 自動化 / Codex
- 件名: CodexプロンプトGit操作準拠チェッカーを実装する
- 状態: 確認待ち
- 優先度: 高
- 実施主体: ChatGPT(リポジトリ編集)
- 関連箇所: CodexPrompt_GitScriptEscalation_Rules, Codex連携運用ルール, `scripts/`, AI運用作業スクリプト化候補棚卸し

## 発端
TSK-029 の棚卸しにより、Codexプロンプトが `CodexPrompt_GitScriptEscalation_Rules.md` に準拠しているかを静的検査する仕組みが、最初に実装すべき候補として整理された。

過去に、Codexプロンプト内で直接 `git checkout` 相当の操作を促す指示が含まれ、`.git/index.lock` 生成に伴う権限昇格再実行へ進みかける問題が発生した。

## 目的
Codex向けプロンプト内に、通常コマンドで直接実行すべきでないGit書き込み操作や、権限昇格再実行を誘発する表現が含まれていないかを静的に検査する。

## 想定検査項目

- `git checkout`
- `git switch`
- `git add`
- `git commit`
- `git push`
- `git merge`
- `git rebase`
- `gh pr create`
- `.git` 直接操作
- 同一Git操作の権限昇格再実行を促す表現
- `CodexPrompt_GitScriptEscalation_Rules.md` の読込・適用指示の有無

## 実装方針

初版は Python CLI として実装した。

想定コマンド例:

    python scripts/check_codex_prompt_git_safety.py CodexExec.md

スクリプト雛形を含むプロンプトでは、必要に応じて以下を使う。

    python scripts/check_codex_prompt_git_safety.py CodexExec.md --allow-script-blocks

チェッカー本体の回帰テストは以下で実行する。

    python scripts/test_codex_prompt_git_safety.py

出力は `PASS` / `FAIL` とし、禁止候補が見つかった場合は行番号、検出語、理由、代替方針を表示する。

## 実施内容

- `scripts/check_codex_prompt_git_safety.py` を追加。
- `CodexPrompt_GitScriptEscalation_Rules.md` の読込・適用指示不足を検査対象に追加。
- 直接Git書き込み操作、PR作成、`.git`直接操作、権限昇格再実行を誘発する表現を検査対象に追加。
- 安全な禁止文が `ELEVATION_RETRY` として誤検知される問題を修正。
- `scripts/test_codex_prompt_git_safety.py` を追加し、正常系・危険系・誤検知防止系・スクリプト雛形許可系の回帰テストを追加。
- `docs/00_プロジェクト管理/05_横断運用規程/CodexプロンプトGit操作準拠チェッカー利用手順.md` を追加・更新。
- PASS確認用の `docs/00_プロジェクト管理/05_横断運用規程/サンプル_Codex安全プロンプト.md` を追加。
- `docs/00_プロジェクト管理/05_横断運用規程/Codex連携運用ルール.md` に、Codex投入前のチェッカー実行導線を追加。

## 利用導線

Codex向けプロンプトをファイルとして作成した場合は、Codexへ投入する前に以下を実行する。

    python scripts/check_codex_prompt_git_safety.py CodexExec.md

スクリプト雛形を含む場合は、必要に応じて以下を使用する。

    python scripts/check_codex_prompt_git_safety.py CodexExec.md --allow-script-blocks

チェッカー本体または検査ルールを変更した場合は、以下の回帰テストを実行する。

    python scripts/test_codex_prompt_git_safety.py

この導線は `Codex連携運用ルール.md` に反映済みである。

## 回帰テスト観点

- 安全なプロンプトが `PASS` になること。
- 必須ポリシー不足が検出されること。
- 必須ポリシー不足を警告扱いにできること。
- 直接Git書き込み指示が `FAIL` になること。
- PR作成指示が `FAIL` になること。
- `.git` 直接操作が `FAIL` になること。
- 権限昇格再実行を許可する表現が `FAIL` になること。
- 権限昇格再実行を禁止する安全文が誤検知されないこと。
- スクリプト雛形内のGit操作が、通常時は検出され、`--allow-script-blocks` 指定時は許可されること。

## AI判断を残す範囲

- Git読み取り操作とGit書き込み操作の文脈上の区別
- 例外的に許容されるスクリプト内操作の判定
- Codexへ渡す作業単位の設計
- チェッカーの検出結果が、文脈上許容される例示かどうかの最終判断
