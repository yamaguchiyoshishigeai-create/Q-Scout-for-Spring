# TSK-024 仕様書ファイル名の日本語統一と命名規約を整備する

- 区分: docs / 運用
- 件名: 仕様書ファイル名の日本語統一と命名規約を整備する
- 状態: 未解決
- 優先度: 高
- 実施主体: ChatGPT(会話内) → ChatGPT(リポジトリ編集) / Codex
- 関連箇所: `docs/`, 仕様書群, AI生成文書, 文書命名規約, docs運用規程

## 発端
AI生成ファイルのファイル名に英語表記の仕様書名が散見され、仕様書群としての体裁・一覧性・見た目の一貫性を乱している。仕様書群は日本語名を中心に整理しているため、仕様書ファイル名を日本語に統一し、今後のブレを防ぐための規約として明文化する必要がある。

## 現状認識
現行docs配下には、日本語ファイル名の仕様書・運用規程が多く存在する一方で、AI生成由来と思われる英語表記ファイル名や英語混在ファイル名が一部残っている可能性がある。

一般的な外部ドキュメント規約では、互換性や検索性の観点からASCII・小文字・ハイフンを推奨する例もあるが、本リポジトリでは既存の仕様書群の体裁と日本語正本の一貫性を優先する。

## 対応方針
以下を実施する。

1. `docs/` 配下の仕様書・設計書・運用規程・AI生成文書を棚卸しする。
2. 英語表記ファイル名または英語混在ファイル名のうち、日本語仕様書群として不自然なものを抽出する。
3. 日本語ファイル名へリネームする候補一覧を作成する。
4. リネーム時は、`README.md`、`docs/README.md`、関連仕様書、改善タスク、PR本文等のリンク切れを防ぐ。
5. docs運用規程または文書命名規則に、以下の厳守仕様を明記する。

## 厳守仕様案

- 仕様書、設計書、運用規程、改善タスク関連の正式文書ファイル名は、原則として日本語名とする。
- AIが新規作成する正式文書ファイル名も、日本語名を原則とする。
- 英語ファイル名は、技術上・慣例上・外部互換上の合理的理由がある場合に限る。
- `README.md`、`README.en.md`、`LICENSE`、`.github/` 配下、設定ファイル、ソースコード、テンプレート、静的資産など、慣例上英語名またはASCII名が妥当なファイルは対象外とする。
- 英語版文書を作る場合も、日本語正本との関係を明示し、英語ファイル名が仕様書群の正本名として混入しないようにする。
- ファイル名変更時は、参照リンク更新を同一PRで行う。

## 完了条件
- docs配下の英語表記仕様書ファイル名が棚卸しされる。
- 日本語名へ統一すべき対象がリネームされる、または対象外理由が整理される。
- 仕様書ファイル名は日本語名を原則とする規約が、docs運用規程または文書命名規則へ明記される。
- 主要README・docs索引・関連文書のリンク切れがない状態になる。
