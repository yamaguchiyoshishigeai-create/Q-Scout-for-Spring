# TSK-017 Spring Boot 3.5.13 merge後の検証を実施する

- 区分: テスト / 依存更新
- 件名: Spring Boot 3.5.13 merge後の検証を実施する
- 状態: 未解決
- 優先度: 最優先
- 実施主体: Codex
- 関連箇所: `pom.xml`, Java CI, Web起動, CLI起動, 主要画面導線

## 発端
PR #41 により Spring Boot parent を 3.3.4 から 3.5.13 へ更新し、main へ merge した。PR上ではCI確認を前提に merge したが、Public Free版としては merge 後の本線検証、Web主要導線、CLI導線の確認を明示的に残す必要がある。

## 現状認識
Spring Boot 3.5.13 化は Public Free版の方針に沿った段階更新である。一方で、`mvnw test`、package、Web起動、主要導線確認、CLI導線確認を一連の完了条件としてまだ台帳化していなかった。

## 対応方針
Codex に main 最新状態での検証を依頼し、以下を確認する。

- `mvnw test`
- `mvnw -q -DskipTests package`
- Web起動
- `/`
- `/help`
- ルール詳細ページ
- ZIPアップロード
- レポート preview
- レポート download
- `?lang=ja` / `?lang=en`
- CLI起動導線

## 完了条件
main 最新状態で Spring Boot 3.5.13 化後の主要検証が完了し、結果が `CodexExec.result` または PR / docs へ記録される。失敗がある場合は、修正タスクまたは個別不具合タスクへ分解される。
