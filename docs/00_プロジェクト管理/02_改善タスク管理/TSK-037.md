- 件名: docsリンク・配置規程チェッカーを実装する

## 発端
TSK-029の棚卸しで、Markdownリンク切れ、docs配置規程違反、索引から辿れない文書、docs直下への不適切配置を検査する候補が抽出された。

TSK-030〜TSK-032が解決済みになったため、TSK-033の残件登録ゲートにより本候補を後続TSKとして登録する。

## 目的
docs配下のリンク・配置・索引到達性を機械的に検査し、文書肥大化、リンク切れ、配置ルール逸脱を早期検出する。

## 想定検査項目

- Markdown相対リンクの存在確認
- docs配下のリンク切れ検出
- docs直下の不適切配置候補検出
- docs索引または関連正本から辿れない文書の検出
- 0Byte文書、プレースホルダ文書候補の検出
- 解決済みTSKリンクの存在確認

## 実装方針

初版は Python CLI を第一候補とする。

将来的には GitHub Actions により、docs変更PRで自動検査することを検討する。

## AI判断を残す範囲

- 文書の置き場所の意味的妥当性
- 新規分類の必要性
- 正本、補助、履歴資料の意味判断
- リンク切れに見える意図的な外部参照の扱い
