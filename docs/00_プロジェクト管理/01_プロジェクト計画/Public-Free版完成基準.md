# Public Free版完成基準

## 1. 文書目的

本書は、`Q-Scout-for-Spring` の Public Free 版を「未完成の公開物」ではなく、無料で試せる範囲が明確に動作する試供版兼ポートフォリオとして扱うための完成基準を定義する。

本書の目的は、以下を明確にすることである。

- Public Free 版として必ず成立しているべき範囲
- Public Free 版では対象外または Private Pro 版へ回す範囲
- README、docs、Web UI、CI、サンプル評価、レポート品質の確認観点
- 今後の改善タスクの優先度判断基準

## 2. 位置づけ

本書は、`Public-Free版・Private-Pro版分離運用方針.md` を補完する完成判定用の基準文書である。

- 分離運用方針: Public Free 版と Private Pro 版の境界を定義する
- 本書: Public Free 版として公開可能な完成状態を定義する

本書は、現行実装の範囲を前提とする。未実装機能を完成済みとして扱わず、現行 README、docs、実装、テスト、公開サンプル評価の到達点を基準に判定する。

## 3. 完成基準の基本原則

Public Free 版の完成基準は、以下の原則で判断する。

1. 無料で試せる範囲が明確であること。
2. 対象外の構成や Pro 版に回す範囲が明確であること。
3. Web UI、CLI、Markdown 成果物の主要導線が破綻していないこと。
4. Spring Boot 3.5.x 系までの安定対応を Public Free 版の上限として扱うこと。
5. 基本6ルールの価値が利用者に伝わること。
6. README と docs から利用方法、制約、位置づけへ辿れること。
7. 公開サンプル評価または自己分析により、利用イメージを説明できること。
8. 機密コードを公開環境へアップロードしない注意喚起が明確であること。
9. AI 非依存の品質診断基盤として価値が成立していること。
10. Pro 版相当の高度機能を Public 側へ入れすぎないこと。

## 4. 必須機能の完成基準

### 4.1 Web UI

Public Free 版では、Web UI について以下を満たすことを完成基準とする。

- トップページから診断実行の目的と使い方が分かる。
- zip アップロード導線が存在する。
- 20MB 制限、60秒制限、公開環境へのアップロード注意が理解できる。
- 解析成功時に総合スコア、重大度別件数、成果物導線が表示される。
- 人間向け Markdown と AI 入力 Markdown の preview / download 導線が存在する。
- help ページとルール詳細ページへ辿れる。
- 日本語 / 英語切替が主要画面で破綻していない。

### 4.2 CLI

Public Free 版では、CLI について以下を満たすことを完成基準とする。

- README に CLI 起動方法が記載されている。
- Web 既定 jar から CLI main を明示起動できる導線が説明されている。
- `run-cli.bat` または同等の補助導線が説明されている。
- 自己分析またはサンプル分析により、CLI 成果物生成の確認が可能である。

### 4.3 解析対象

Public Free 版の主対象は、以下に限定してよい。

- Spring Boot 3.5.x 系まで
- Java 17 / 21
- Maven 単一モジュール
- Java ソースコードを中心とする Spring Boot プロジェクト
- zip の直下または単一トップディレクトリ配下に `pom.xml` を持つ構成

以下は、完成基準上の必須対応に含めない。

- Spring Boot 4.x
- Spring Framework 7
- Gradle
- マルチモジュール
- Kotlin 主体プロジェクト
- 複数リポジトリ横断分析
- 大規模案件向け非同期ジョブ管理

### 4.4 診断ルール

Public Free 版では、基本6ルールが診断価値の中核である。

完成基準は以下とする。

- 基本6ルールが README または docs から理解できる。
- ルール詳細ページまたはレポートから、各ルールの意味が利用者に伝わる。
- ルール結果がスコア、重大度、違反一覧、改善ヒントへ反映される。
- ルール精度改善余地がある場合も、候補検出型の診断であることが説明されている。

### 4.5 Markdown 成果物

Public Free 版では、以下の2成果物を完成基準とする。

- `qscout-report.md`
- `qscout-ai-input.md`

人間向けレポートについては、以下を満たすこと。

- 総合スコアが確認できる。
- 違反一覧が確認できる。
- 重大度またはルール別の概要が確認できる。
- 改善の出発点が分かる。
- 日本語 / 英語の利用導線と大きく矛盾しない。

AI 入力 Markdown については、以下を満たすこと。

- AI へ貼り付けて相談しやすい構造になっている。
- 人間向けレポートとは用途が分離されている。
- 現行方針どおり、英語固定であることが説明可能である。

## 5. README / docs の完成基準

### 5.1 README

README は、Public Free 版の入口として以下を満たすこと。

- Q-Scout for Spring の目的が冒頭で分かる。
- Public Free 版としての位置づけが明記されている。
- build、Web 起動、CLI 起動、補助スクリプトが分かる。
- 公開サンプル評価または自己分析への導線がある。
- Free 版の主対象と Pro 版へ回す領域が過不足なく説明されている。

### 5.2 docs

`docs/` は、Public Free 版の判断根拠として以下を満たすこと。

- `docs/README.md` から正本・補助仕様・履歴資料の位置づけが分かる。
- `00_プロジェクト管理/` 配下に管理系文書が整理されている。
- `10_企画/`、`20_要件定義/`、`30_基本設計/`、`40_詳細設計/` の役割が分かる。
- Public Free 版 / Private Pro 版の分離方針へ辿れる。
- 本書の完成基準へ辿れる。

## 6. 検証の完成基準

Public Free 版では、以下の検証を完了候補として扱う。

### 6.1 最小必須検証

- `mvnw test` または同等の test 実行
- `mvnw -q -DskipTests package` または同等の package 実行
- Web 起動確認
- トップページ表示確認
- help ページ表示確認
- ルール詳細ページ表示確認
- zip アップロード解析確認
- preview / download 導線確認
- `?lang=ja` / `?lang=en` の主要導線確認
- CLI 起動導線確認

### 6.2 記録基準

検証結果は、以下のいずれかへ記録する。

- PR 本文
- PR コメント
- コミットメッセージ本文の短縮記録
- 必要に応じて docs の検証メモ

長大ログやローカル実行結果ファイルは、原則として git 管理しない。

## 7. サンプル評価の完成基準

Public Free 版では、公開サンプル評価を「商用品質の網羅検証」ではなく、Free 版の対応範囲と限界を説明するための補助導線として扱う。

完成基準は以下とする。

- 中核候補、補助候補、重量級候補、保留候補の分類が説明されている。
- 成功例だけでなく、失敗例や保留例の理由も記録できる。
- Maven 単一モジュールを主対象とする Free 版の制約と矛盾しない。
- 保留候補を無理に成功扱いしない。

## 8. セキュリティ・公開環境注意の完成基準

Public Free 版は公開環境で試される可能性があるため、以下を完成基準とする。

- 機密情報、顧客コード、社外秘ソースをアップロードしない注意喚起がある。
- アップロード対象が zip であることが明確である。
- サイズ制限と実行時間制限が明確である。
- 一時ファイル保持や成果物期限切れの考え方が利用者に誤解されにくい。

## 9. Pro版へ回すべき項目

以下は Public Free 版の完成条件に含めない。

- Spring Boot 4.x 対応
- Spring Framework 7 対応
- Java 25 対応
- Gradle 対応
- マルチモジュール対応
- Kotlin 対応
- CI/CD 連携
- ルールカスタマイズ
- 診断履歴管理
- 組織向けレポート
- 誤検知抑制設定
- 大規模プロジェクト向け非同期解析
- 商用向け互換性マトリクス

ただし、Free 版の安全性、説明性、基本品質を高める基盤修正は、Pro 版の検証から得た知見であっても、機能境界を守ったうえで Public 側へ還元してよい。

## 10. 完成判定チェックリスト

Public Free 版の完成判定では、以下を確認する。

| 区分 | 確認項目 | 判定 |
|---|---|---|
| README | 目的、対象、起動方法、Free版位置づけが説明されている | 必須 |
| docs | 正本、補助仕様、履歴資料、完成基準へ辿れる | 必須 |
| Web | アップロード、結果表示、preview、download が成立する | 必須 |
| CLI | CLI 起動方法と成果物生成導線が説明されている | 必須 |
| i18n | 日本語 / 英語の主要導線が破綻していない | 必須 |
| レポート | 人間向け / AI向け成果物が分離されている | 必須 |
| 制約 | 20MB、60秒、zip、Maven単一モジュール前提が明示されている | 必須 |
| セキュリティ | 公開環境へ機密コードをアップロードしない注意がある | 必須 |
| サンプル | 公開サンプル評価の分類と限界が説明されている | 推奨 |
| CI / test | main 最新で最小検証結果が記録されている | 必須 |
| Pro分離 | Pro版へ回す範囲がPublic側に混入していない | 必須 |

## 11. 改善タスク優先度への適用

今後の改善タスクは、本書を基準に以下の優先度で判断する。

1. Public Free 版の必須導線を壊しているもの
2. Spring Boot 3.5.x 系までの安定動作確認に関わるもの
3. README / docs から利用者が迷う原因になるもの
4. 公開環境上の安全性・誤解防止に関わるもの
5. レポート品質と基本6ルールの説明性に関わるもの
6. ポートフォリオ訴求を高めるもの
7. Pro 版構想に関わるもの
8. 保留中の追加 UI 調整や高度化項目

## 14. 最終結論

Public Free 版は、Pro 版の廉価版として機能を詰め込むものではなく、Q-Scout for Spring の基本価値を安全に試せる公開版として完成させる。

その完成状態は、Web / CLI / Markdown 成果物 / README / docs / 検証記録 / サンプル評価 / 公開環境注意の各観点から判断する。

本書を基準に、今後の改善タスクは「Public Free 版の完成に必要なもの」と「Private Pro 版へ回すもの」を切り分けて進める。
