# Codex連携運用ルール

## 1. 文書目的

本書は、ChatGPT が Codex 向け作業指示を作成し、Codex 実行結果を確認する際に参照すべきルール文書の入口である。

Codex 関連作業では、原則として本書を最初に読み、下記の関連ルールに従う。

---

## 2. 適用対象

本書は、主に以下の場面に適用する。

- ChatGPT が Codex 向けプロンプトを作成する場合
- Codex にリポジトリ上の実装、docs 更新、テスト、build、PR 作成を依頼する場合
- Codex 実行後に PR 本文、PR コメント、コミットメッセージ本文、必要に応じてローカルの `CodexExec.result` を確認する場合
- Codex 関連の追加運用ルールを整理する場合

Git 書き込みを伴わない設計相談、文案作成、レビューのみの場合でも、Codex に作業を依頼する可能性があるときは本書を入口として確認する。

---

## 3. 必ず参照する関連ルール

### 3.1 Codex プロンプト作成時

Codex にリポジトリ作業を依頼するプロンプトを作成する場合は、次を参照する。

- `CodexPrompt_GitScriptEscalation_Rules.md`

特に、Git 書き込み操作、branch 作成、commit、push、PR 作成を Codex の通常コマンド経路で直接実行させず、専用 `ps1` / `sh` スクリプト経由に限定する。

### 3.2 Codex 実行結果確認時

Codex 作業結果、commit、push、PR を確認する場合は、次を参照する。

- `AIリポジトリ作業証跡管理ルール.md`

特に、Codex 固有の実行結果確認も本統合ルール内の Codex 節に従い、PR がある場合は PR 本文、PR コメント、PR 差分、コミットメッセージ本文を確認する。旧運用履歴が必要な場合のみ、`docs/90_アーカイブ/` の旧ログ・旧規程を参照する。

Codex が PR を作成し、ユーザー確認またはローカル確認が必要な場合の merge 前確認も、`AIリポジトリ作業証跡管理ルール.md` の PR ブランチ検証先行ルールを正本とする。本書では入口として参照先を示し、具体的な確認順序・例外条件・証跡記録先は同正本に集約する。

## 4. 標準的な読み順

Codex 関連作業では、原則として以下の順に確認する。

1. `Codex連携運用ルール.md`
2. Codex プロンプト作成が必要な場合は `CodexPrompt_GitScriptEscalation_Rules.md`
3. Codex 実行結果確認または PR ブランチ検証が必要な場合は `AIリポジトリ作業証跡管理ルール.md`
4. PR 本文 / PR コメント / PR 差分 / コミットメッセージ本文
5. 旧運用履歴が必要な場合のみ `docs/90_アーカイブ/` を参照

## 5. ChatGPT への標準指示

Codex 向けプロンプト作成を依頼する場合は、原則として次の指示で足りる。

```text
Codex連携運用ルール.md を読み、そこに記載された関連ルールに従って Codex 向けプロンプトを作成してください。
```

Codex 実行結果確認を依頼する場合は、原則として次の指示で足りる。

```text
Codex連携運用ルール.md を読み、AIリポジトリ作業証跡管理ルール.md に従って、PR本文、PRコメント、コミットメッセージ本文、関連PR差分を確認してください。
```

Codex 作成 PR の merge 前確認を依頼する場合も、`AIリポジトリ作業証跡管理ルール.md` の PR ブランチ検証先行ルールに従う。

## 6. 作業主体の整理

Codex 関連作業では、現行の作業主体分類に従う。

| 作業主体 | 主な担当 |
|---|---|
| ユーザー(人) | 最終判断、外部確認、main への merge、主観的判断 |
| ChatGPT(会話内) | 方針整理、レビュー、Codex 向けプロンプト作成 |
| ChatGPT(リポジトリ編集) | 小〜中規模ファイルの局所編集、README / docs の軽微更新 |
| Codex | 実装、複数ファイル更新、テスト、build、PR 準備 |

作業仕分け時は、対象ファイルの規模、構造複雑度、全文取得可否、テスト / build の要否を確認する。

### 6.1 Codex節約型作業分担ルール

Codex の使用量が逼迫している場合、またはユーザーが Codex 温存方針を示した場合は、通常よりも厳格に作業を細分化し、Codex へ渡す範囲を最小化する。

Codex に依頼する前に、必ず次を確認する。

1. ChatGPT(会話内) で方針整理、比較検討、作業分解、Codex 指示書作成まで完了できないか。
2. ChatGPT(リポジトリ編集) で docs、README、改善タスク台帳、小規模設定変更、単一〜少数ファイルの局所修正を先に処理できないか。
3. Codex に渡す場合でも、広範囲一括依頼ではなく、最小検証・最小修正単位に分割できないか。
4. Codex 実行後に発生した追加課題を改善タスク台帳へ登録する必要がないか。

Codex へ優先的に渡す作業は、原則として以下に限定する。

- build / test 実行
- Web 起動確認
- CLI 実行確認
- 実行ログの取得・切り分け
- 多数ファイルの一括リネーム
- リンク総点検
- 広範囲 grep を伴う確認
- 反復修正や実行環境での検証が必要な実装修正

Codex 節約型運用中は、Codex 向けプロンプトに「実施対象」「実施しないこと」「失敗時に修正せず記録する条件」「最終出力形式」を明示し、不要な探索や大規模修正を抑制する。

## 7. 追加ルール発生時の扱い

Codex 関連の新ルールを追加する場合は、原則として個別ルールファイルとして作成し、本書の参照一覧へ追記する。

ただし、既存ルールの軽微な補足で済む場合は、既存ファイルへ追記してよい。

### 7.1 個別ファイル化を優先する例

- Git 操作、PR 作成、権限付き実行など、独立した安全制約を持つルール
- AI リポジトリ作業の証跡記録先や確認順序を定義するルール
- 将来的に単独参照される可能性が高いルール

### 7.2 既存ファイル追記でよい例

- 既存ルールの用語補足
- チェックリストの軽微な追加
- 既存手順の例示追加

---

## 8. 運用上の注意

- 本書は入口文書であり、詳細ルールの重複記載を避ける。
- 詳細な禁止事項、スクリプト雛形、ログ形式、PR ブランチ検証先行ルールは各関連ルール文書を正本とする。
- Codex 向けプロンプトを作成する際は、Git 操作スクリプト昇格ゲート方式を前提とする。
- Codex 実行結果を確認する際は、`AIリポジトリ作業証跡管理ルール.md` を参照し、統合ルール内の Codex 節に従って確認先を切り替える。
- Codex 作成 PR の merge 前確認が必要な場合は、`AIリポジトリ作業証跡管理ルール.md` の PR ブランチ検証先行ルールに従う。
- 旧ルールを個別に毎回列挙するのではなく、原則として本書を入口に関連ルールを辿る。
- Codex 節約型運用中は、Codex へ渡す前に ChatGPT(会話内) と ChatGPT(リポジトリ編集) で切り出せる作業がないかを必ず確認する。
