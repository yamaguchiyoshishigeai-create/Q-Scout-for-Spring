# 実装系文書配置規程

## 1. 目的

本規程は、Q-Scout for Spring における実装系文書の配置基準を定めるための正式規程である。

本規程の目的は、`docs/40_詳細設計/44_実装指示書/` と `docs/50_実装/` の役割重複や誤配置を防ぎ、あわせて docs 直下への規程文書乱立を防止することにある。

特に、Codex 等へ渡す差分実装依頼文書と、実装工程そのものに継続適用する正式文書とを明確に分離し、将来的な `docs/60_単体テスト/`、`docs/70_結合テスト/` を含む工程文書整備時にも判断がぶれない状態をつくる。

## 2. 適用範囲

本規程は、実装に関連する以下の文書に適用する。

- Codex 等へ渡す実装指示書
- 差分実装依頼文書
- 実装標準
- 実装規約
- 実装工程運用ルール
- 実装完了条件

なお、設計判断そのものを定義する文書は、引き続き `docs/40_詳細設計/` を正本とし、本規程は実装系文書の置き場と運用境界を定めるものとして扱う。

## 3. 基本原則

### 3.1 単一作業文書と恒久文書の分離

特定の作業、差分、PR、修正テーマの実行を目的とする文書と、複数作業に継続適用する恒久文書とは分離する。

単一作業を進めるための文書を恒久文書として扱うと、作業終了後に参照先が曖昧になる。一方、恒久文書を作業文書置き場へ混在させると、後続作業で正式な基準を見失いやすくなる。

### 3.2 設計文書と実装文書の分離

実装系文書は、設計文書と混同しない。画面差分、責務分割、部品仕様等の設計判断を定義する文書は `docs/40_詳細設計/` に置き、設計を受けて実装をどう実行するか、どの基準で統制するかを扱う文書を本規程の対象とする。

### 3.3 置き場所に迷ったことを理由に配置しない

文書の主目的が曖昧なまま配置先を決めてはならない。特に、置き場所に迷ったことを理由に `docs/` 直下へ規程文書を追加することを禁止する。

docs 全体または複数工程に横断して効く規程は、`docs/40_詳細設計/44_実装指示書/` や `docs/50_実装/` ではなく、まず `docs/00_プロジェクト管理/05_横断運用規程/` を検討する。

## 4. 配置先の定義

### 4.1 `docs/40_詳細設計/44_実装指示書/` の定義

`docs/40_詳細設計/44_実装指示書/` は、設計に基づいて特定の実装作業を実行するための作業用文書置き場である。

ここには、Codex 向け実装指示書、差分単位の修正依頼、特定 PR や特定タスクの実行文書など、単一の作業、差分、PR、修正テーマに閉じている文書を置く。

これらの文書は、作業完了後に主参照先でなくなってもよい。すなわち、`44_実装指示書/` は継続基準の保管場所ではなく、実装を動かすための作業文書の配置先である。

### 4.2 `docs/50_実装/` の定義

`docs/50_実装/` は、実装工程そのものに継続適用する正式文書置き場である。

ここには、実装規約、実装標準、実装完了条件、実装工程の正式運用ルールなど、単一タスクに閉じず複数作業に横断適用する文書を置く。個別作業の依頼文書や単発の差分修正メモはここに置かない。

現時点で該当する正式文書が十分に整っていない場合でも、無理に既存文書を `50_実装/` へ移設しない。まずは適切な配置基準を定め、新規文書追加時と将来の棚卸し時に整合を取る。

### 4.3 `docs/00_プロジェクト管理/05_横断運用規程/` の定義

`docs/00_プロジェクト管理/05_横断運用規程/` は、docs 全体または複数工程に横断して適用する正式な運用規程文書の集約先である。

たとえば、実装系文書の配置規程、将来的な docs 作成運用方針書、将来的な文書命名規則、将来的な `CodexExec.result` 関連運用規程など、特定工程や単一作業に閉じない規程は、まずこの配下を検討する。

本規程自身も、`44_実装指示書/` や `50_実装/` だけに効く文書ではなく、docs 全体の配置判断へ影響する横断規程であるため、この配下に置く。

## 5. 配置判断手順

実装系文書を新規追加する際は、少なくとも次の順で判断する。

1. 特定作業や差分を実行するための文書か。該当する場合は `docs/40_詳細設計/44_実装指示書/` に置く。
2. 実装工程に継続適用する正式基準文書か。該当する場合は `docs/50_実装/` に置く。
3. docs 全体または複数工程に横断適用する規程か。該当する場合は `docs/00_プロジェクト管理/05_横断運用規程/` に置く。

この 3 段階で判断しても位置づけが定まらない場合は、文書の主目的を先に言語化し、そのうえで `docs/README.md` や関連ガイド文書の更新を伴って整理する。

## 6. 禁止事項

以下を禁止する。

- 恒久的な実装標準文書を `docs/40_詳細設計/44_実装指示書/` に置くこと
- 単一作業向け実装依頼文書を `docs/50_実装/` に置くこと
- 配置先が曖昧なまま文書を追加すること
- docs 直下へ、置き場所に困った規程文書を追加すること

## 7. 当面の運用方針

本規程の採用後も、既存文書をただちに全面移設するものではない。

当面は、まず本規程を配置基準として採用し、新規追加文書や更新文書の判断に反映する。必要が生じた時点で、既存の `docs/40_詳細設計/44_実装指示書/` 配下文書を棚卸しし、作業文書と恒久文書の切り分けを段階的に進める。

また、`docs/50_実装/`、`docs/60_単体テスト/`、`docs/70_結合テスト/` は将来的な工程文書の正式配置先として維持するが、現時点で実体が薄いことを理由に役割を曖昧化しない。

## 8. 最終原則

最終原則は、以下のとおりである。

- `docs/40_詳細設計/44_実装指示書/` は、実装を動かすための文書である。
- `docs/50_実装/` は、実装を統制するための文書である。
- docs 全体または複数工程に横断して効く規程は、`docs/00_プロジェクト管理/05_横断運用規程/` に集約する。

以後、実装系文書の配置判断は、この原則を起点に行う。
