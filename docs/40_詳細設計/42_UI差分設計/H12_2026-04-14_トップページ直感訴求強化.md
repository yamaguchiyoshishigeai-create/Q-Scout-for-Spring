# トップページ直感訴求強化

## 位置づけ

本書は、2026-04-14 に、トップページの直感訴求強化と見せ方改善の意図を詳細設計として残すために作成された文書である。

## 提案名
**トップページ直感訴求強化**

## 1. 文書目的
本差分仕様書は、v1.2 までで再構成されたトップページに対し、**直感的理解を阻害している説明過多状態を改善する** ための追加仕様を定義するものである。

目的は以下の3点である。

1. トップページ冒頭だけで「何のシステムか」「どう使うか」「何が返るか」が分かるようにする  
2. 成果物サンプルをテキスト中心から画像中心へ置き換え、視認性を高める  
3. Markdown 成果物の見本提示を、長文抜粋ではなく preview 画面への導線中心へ整理する  

---

## 2. 差分の背景
v1.2 では `#intro` セクション内の成果物サンプルが強化され、スコア系サンプルと Markdown 系サンプルの識別性は向上した。  
しかし実際の画面では、成果物サンプル説明が依然として文章依存であり、トップページとして必要な **“一目で分かる”** 状態にはまだ達していない。

特に以下の問題がある。

- 冒頭説明が技術構造寄りで、何のシステムかの断言が弱い
- 成果物サンプルが文章量過多で視覚負荷が高い
- Markdown の短い抜粋を読ませる構造が、トップページの理解導線として重い
- 実際には結果サマリー画像の方が、品質診断ツールとしての価値を伝えやすい

そのため v1.3 では、**トップページの役割を「読ませる」から「見せる」へ寄せる**。

---

## 3. 差分の基本方針

### 方針A：v1.2 のレイアウト構造は維持する
左サイドバー、2カラム、各セクション構成は維持する。  
今回の差分は情報の見せ方の改善であり、ページ構造の再設計ではない。

### 方針B：冒頭説明は短く断定的にする
冒頭では技術説明より先に、**何のシステムか** を一文で明示する。

### 方針C：成果物サンプルの主役を画像にする
テキストサンプルより、結果サマリーの縮小画像を主サンプルとして採用する。

### 方針D：Markdown 成果物は導線中心にする
Markdown の長文抜粋はトップから外し、  
**「出力成果物はこちら」→ preview サンプル画面へ誘導** する構成へ寄せる。

---

## 4. 差分対象

### 主対象
- `#intro` セクション
- hero の冒頭説明文
- 成果物サンプル表示部

### 補助対象
- `messages_ja.properties`
- `messages_en.properties`
- 必要ならトップページ描画確認テスト

### 原則非対象
- `#artifacts` セクションの大幅改修
- help ページ
- preview ページ本文
- 左サイドバー構造
- 結果サマリの実データ表示部分
- 解析ロジック

---

## 5. 追加仕様

## 5.1 冒頭説明文の再設計
現行の長い hero lead を、以下の趣旨へ差し替える。

### 表示方針
**1行目は強い断定文**
- 少し大きめ、または強調表示

**2行目以降は通常文**
- 使い方
- 出力結果
を簡潔に説明

### 推奨日本語文案
**このシステムは、Spring Boot / Maven プロジェクトの品質診断システムです。**  
Web版では、20MB以下のZIPファイルをアップロードして診断を実行できます。  
解析結果は、ユーザー向けレポートとAI向け入力の2種類で確認・ダウンロードできます。

### 英語文案方針
- 簡潔
- 断定的
- 技術的だが読みやすい

---

## 5.2 成果物サンプルを画像中心へ変更
`#intro` の成果物サンプル表示部は、テキスト中心から**結果サマリー画像中心**へ変更する。

### 必須要件
- 結果サマリーの縮小画像を表示する
- 画像はトップ上で視認可能なサイズにする
- 画像は「実行するとこのような結果が返る」ことを直感的に示すものとする

### 画像対象
優先度順は以下。

1. 結果サマリー画面の縮小画像
2. 必要なら人間向けMarkdown preview の縮小画像
3. 必要ならAI入力Markdown preview の縮小画像

ただし、トップ上で情報が多すぎる場合は、  
**結果サマリー画像1枚 + Markdown導線2本** を優先する。

---

## 5.3 Markdown サンプルは導線中心へ変更
v1.2 の Markdown 抜粋テキストは削減または廃止する。

### 新方針
トップでは以下の形に寄せる。

- ユーザー向けレポートのサンプルを見る
- AI向け入力のサンプルを見る

必要に応じて小さなサムネイル画像を添えてよいが、  
長いテキスト抜粋は置かない。

### 役割整理
- トップ = 見本と導線
- preview = 実際の中身確認

---

## 5.4 成果物サンプル部の構成
### 推奨構成
- サンプル見出し
- 短い説明文
- 結果サマリー画像
- サンプル導線2本
  - ユーザー向けMarkdownのサンプルを見る
  - AI入力Markdownのサンプルを見る

### トップで伝えたいこと
- 診断後にスコアと違反件数が返る
- 成果物は人向けとAI向けに分かれている
- どちらもプレビューとダウンロードが可能

---

## 5.5 `#artifacts` セクションとの役割分離
`#artifacts` は引き続き、

- 何の成果物か
- preview / download 導線
を説明する場所とする。

一方、`#intro` は、

- 何が返るか
- それがどう見えるか
を短く見せる場所とする。

### 役割整理
- `#intro` = **直感訴求**
- `#artifacts` = **説明と操作導線**

---

## 5.6 レイアウト方針
### デスクトップ
- 結果サマリー画像を見やすく配置する
- 画像の横または下に短い導線を置く
- テキスト量は減らす

### モバイル
- 画像は横幅いっぱいに近い形で縮小表示
- 導線ボタンは縦積みでよい
- テキスト抜粋は極力削る

---

## 5.7 テスト方針
v1.3 では、主要文言と導線の存在確認を中心に維持する。

### 必須確認
1. 日本語トップで新しい冒頭説明が表示される
2. 英語トップで対応する説明が表示される
3. 結果サマリー画像または画像導線が表示される
4. ユーザー向け / AI向けのサンプル導線が表示される
5. `response == null` / `response != null` を壊していない

---

## 6. 今回やらないこと
- 結果サマリー画像の動的生成
- 画像ギャラリー化
- preview ページの全面刷新
- 詳細違反一覧のトップ常設
- 履歴比較
- グラフ追加
- help ページ再設計

---

## 7. 完了条件
以下を満たしたら v1.3 差分実装完了とする。

1. 冒頭で「何のシステムか」が一文で伝わる  
2. 成果物サンプルの主役が画像になっている  
3. Markdown サンプルが導線中心に整理されている  
4. トップが以前より“見て分かる”構成になっている  
5. 左サイドバー構造を壊していない  
6. 日本語 / 英語切替を維持している  
7. モバイルで破綻していない
