# 左サイドバー前提トップページ再構成 差分デザイン仕様書 v1.2

## 提案名
**左サイドバー前提トップページ再構成 差分デザイン仕様書 v1.2**

## 1. 文書目的
本差分仕様書は、v1.1 で再構成されたトップページに対し、**「トップページ成果物サンプル追加案 v1.0」** の意図をより明確に反映するための追加仕様を定義するものである。

目的は以下の3点である。

1. 実行前ユーザーが、Q-Scout で得られる成果物を直感的に理解できるようにする  
2. `#intro` セクションにおける成果物サンプルの情報密度と識別性を高める  
3. `#artifacts` セクションを「詳細確認導線」、`#intro` セクションを「価値訴求サンプル」として役割分離する  

---

## 2. 差分の背景
v1.1 実装後のトップページでは、`#intro` セクションに `sample-card` が存在し、成果物サンプルは完全には欠落していない。  
ただし現状のサンプルは、**1つの汎用サンプルブロック** に留まっており、

- 総合スコアのイメージ
- 違反件数 / HIGH・MEDIUM・LOW のイメージ
- ユーザー向けMarkdownの雰囲気
- AI入力Markdownの雰囲気

をそれぞれ分けて認識しにくい。

一方、Q-Scout は品質スコアリングツールであり、MVP でも **総合スコア出力、人間用Markdown出力、AI用Markdown出力** が成功条件に含まれている。  
そのため、成果物サンプルは補助要素ではなく、**トップページの価値理解を支える重要要素** として扱う必要がある。

---

## 3. 差分の基本方針

### 方針A：v1.1 の構造は維持する
左サイドバー、2カラム、`#intro` / `#run-analysis` / `#result-summary` / `#artifacts` の構造はそのまま維持する。  
今回の差分は再構成のやり直しではなく、成果物サンプルの強化である。

### 方針B：成果物サンプルは `#intro` で強化する
成果物の**役割説明と導線** は `#artifacts` に残す。  
一方、成果物の**見本提示と価値訴求** は `#intro` で強化する。

### 方針C：全文ではなく短い比較サンプルにする
preview ページ全文をトップへ埋め込まない。  
サンプルはあくまで「成果物の種類と雰囲気が一目で伝わる」粒度に留める。  
これは単画面 Web UI のシンプルさを崩さないという方針とも一致する。

---

## 4. 差分対象

### 主対象
- `#intro` セクション
- `sample-card` またはそれに相当する成果物サンプル表示部

### 補助対象
- `messages_ja.properties`
- `messages_en.properties`
- 必要ならトップページ描画確認テスト

### 原則非対象
- `#artifacts` セクションの全面改修
- help ページ
- preview ページ
- 解析ロジック
- ダウンロード導線
- locale switcher

---

## 5. 追加仕様

## 5.1 `#intro` セクションにおける成果物サンプルの必須化
v1.1 では `sample-card` は存在するが、v1.2 ではこれを **必須要件** とする。  
`#intro` セクションは、単なるヒーローや概要ではなく、**Q-Scout を使うと何が返ってくるかを視覚的に理解できる場所** として位置づける。

### 必須表示項目
少なくとも以下を含むこと。

1. **総合スコア例**
2. **違反件数と重大度内訳例**
3. **ユーザー向けMarkdownの短い抜粋**
4. **AI入力Markdownの短い抜粋**

---

## 5.2 サンプル構成の再設計
現状の単一 `pre` ブロック型から、**比較可能な複数サンプルブロック型** へ変更する。

### 推奨構成
- サンプル全体タイトル
- サンプル説明文
- 2列または縦積みで以下を表示  
  - スコアサンプル
  - ユーザー向けMarkdown抜粋
  - AI入力Markdown抜粋

### サンプル例
#### スコアサンプル
- Overall Score: 84/100
- Violations: 6
- HIGH=1 / MEDIUM=2 / LOW=3

#### ユーザー向けMarkdown抜粋
- Q-Scout Report
- Overall Score
- Rule Summary

#### AI入力Markdown抜粋
- Project Analysis Input
- Detected Issues
- Suggest improvements

---

## 5.3 `#artifacts` セクションとの役割分離
`#artifacts` セクションは引き続き、

- ユーザー向けMarkdownとは何か
- AI入力Markdownとは何か
- プレビュー / ダウンロード

を示す場所とする。

一方、`#intro` セクションは、

- 実行前ユーザーが成果物の見え方を理解する
- プレビューへ行く前に成果物の違いを理解する

ための場所とする。

### 役割整理
- `#intro` = **価値訴求と見本**
- `#artifacts` = **説明と導線**

---

## 5.4 文言設計方針
多言語化仕様では、Web UI 文言は MessageSource へ集約し、日本語デフォルト・英語併用型で運用する。  
そのため、v1.2 で追加するサンプル見出し・説明文も `page.home.sample.*` 系へ集約する。

### 日本語文言方針
- 直訳調ではなく読みやすさを優先
- 「このツールで何が返るか」が明確に伝わる表現

### 英語文言方針
- 簡潔
- 過度に長文化しない
- 既存AI入力Markdownの英語トーンと衝突しない

---

## 5.5 レイアウト方針
### デスクトップ
- 既存 `hero-grid` を活かしてよい
- `sample-card` 内をサブグリッド化してもよい
- サンプルは 2〜3 ブロックまでに抑える

### モバイル
- サンプルブロックは縦積み
- コード断片全文は出さない
- 情報量を増やしすぎない

---

## 5.6 テスト方針
多言語化仕様では、主要文言の存在確認を中心にテンプレート描画確認を行う方針である。  
そのため v1.2 でも、以下を最低限追加または更新する。

### 必須確認
1. 日本語トップで成果物サンプル見出しが表示される
2. 英語トップで成果物サンプル見出しが表示される
3. スコア例・人間用Markdown例・AI入力Markdown例の主要文言が描画される
4. 既存の `response == null` / `response != null` の挙動を壊していない

---

## 6. 今回やらないこと
- 実データに連動したサンプル自動生成
- preview ページ本文の埋め込み
- 違反一覧のトップ常設表示
- 履歴比較
- グラフ化
- help ページの再設計

---

## 7. 完了条件
以下を満たしたら v1.2 差分実装完了とする。

1. `#intro` セクションで成果物サンプルが強化されている  
2. スコア例、人間向けMarkdown例、AI入力Markdown例が視覚的に区別できる  
3. `#artifacts` セクションの役割と競合していない  
4. 既存の左サイドバー構造を壊していない  
5. 日本語 / 英語切替を維持している  
6. モバイルで破綻していない
