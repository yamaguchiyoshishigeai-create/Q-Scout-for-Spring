# Springコード品質スコアリングAI 企画起案

## 2. B案：Springコード品質スコアリングAI

### 2-1. 目的

Springプロジェクトを解析し、**設計品質・保守性・テスト性・Springらしい実装度**をスコアリングするツールです。

単なる Lint ではなく、以下を対象にします。

- DI設計の妥当性
- 層分離の適切性
- `@Transactional` 適用の妥当性
- 例外設計
- テスト戦略
- 設定の整理度
- Spring idiom準拠度

---

### 2-2. アーキテクチャ設計

#### 全体構成

##### 静的解析層
- AST解析
- Annotation解析
- パッケージ依存解析
- テストコード解析

##### ルール評価層
- Springベストプラクティス判定
- 設計原則判定
- 禁止パターン検出

##### AI補助評価層
- ルールでは表現しにくい実装傾向を自然言語で解説
- 改善提案の生成

##### スコアリング層
- 項目別スコア
- 総合スコア
- リスクレベル
- 改善優先度

##### 出力層
- HTML / Markdownレポート
- CI連携用JSON
- PRコメント向け要約

#### 推奨技術構成

- **Backend**: Spring Boot
- **Static analysis**:
  - JavaParser / Spoon
  - ArchUnit
- **Rule engine**: 独自ルールエンジン
- **LLM連携**: 解説生成専用
- **Integration**: GitHub Actions / CI連携

#### 評価カテゴリ例

- Architecture
- Dependency Injection
- Transaction Management
- Exception Handling
- Test Coverage Readiness
- Configuration Hygiene
- Package Layering
- Spring Convention Adherence

---

### 2-3. MVP機能定義

#### MVP機能
- Controller / Service / Repository層の分離判定
- フィールドインジェクション検出
- `@Transactional` の過不足チェック
- 例外握りつぶし検出
- テストクラス有無と基本構成確認
- パッケージ依存の逆流検出
- 総合スコアと改善提案出力

#### MVPで外す機能
- 実行時メトリクス統合
- 学習型の業界別評価モデル
- チーム基準ごとの動的カスタムルールUI
- 自動修正機能

---

### 2-4. 成功条件

- 典型的なSpringアンチパターンを検出できる
- 項目別スコアが説明可能である
- 改善提案が、開発者にとって具体的である

---

### 2-5. 開発ステップ

#### フェーズ1：評価基準定義

##### STEP B1
- **実行担当**：人間
- **理由**：何を「品質が高い」とみなすかは、組織標準やプロダクト方針に依存するため

**作業**
- 評価カテゴリ定義
- スコア配点決定
- 許容アンチパターン整理

##### STEP B2
- **実行担当**：ChatGPT
- **理由**：評価観点の構造化、ルール文言案、スコア説明文作成に強いため

**作業**
- ルール定義書ドラフト作成
- ルールの重みづけ案
- 検出メッセージ文案作成

#### フェーズ2：解析エンジン実装

##### STEP B3
- **実行担当**：Codex
- **理由**：AST解析・ルール実装・レポート出力のコード化を効率的に進められるため

**作業**
- JavaParserベースの解析器実装
- ルール評価器実装
- スコア計算器実装
- レポート出力実装

##### STEP B4
- **実行担当**：人間
- **理由**：誤検知・過検知の許容判断は現場知見とレビュー経験が必要なため

**作業**
- 実案件コードで試験
- 誤検知分析
- 配点調整
- 運用しきい値設定

#### フェーズ3：MVP提供

##### STEP B5
- **実行担当**：Codex
- **理由**：CI連携・JSON出力・CLI整備などの周辺実装をまとめて進めやすいため

**作業**
- CI連携用出力追加
- CLI引数整理
- テスト拡充
- サンプルプロジェクト追加

##### STEP B6
- **実行担当**：ChatGPT
- **理由**：スコアレポートの読み方、改善提案テンプレート、営業説明資料の叩き台作成に向くため

**作業**
- レポート読解ガイド作成
- 導入提案書ドラフト作成
- ユースケース別説明文作成
