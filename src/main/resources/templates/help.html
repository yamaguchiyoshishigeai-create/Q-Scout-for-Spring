使い方・仕様

対応範囲、解析手順、成果物の確認方法をまとめた固定ページです。

利用の流れ

1. zipをアップロード

Spring Boot / Maven の単一モジュール zip をアップロードします。

2. 結果を確認

結果画面でスコア、違反件数、解析対象ファイルを確認できます。

3. Markdownをプレビュー

ユーザー向けMarkdownとAI入力Markdownは、まずブラウザで内容確認してからダウンロードできます。

現在の対応範囲

Java17 想定
フレームワークSpring Boot
ビルドツールMaven
リポジトリ構成単一リポジトリ
モジュール構成単一モジュール
入力形式zip
必須ファイルルートに pom.xml
ソース条件Java ソースコードを含むこと
最大サイズ20MB

成果物

ユーザー向けMarkdown

診断結果を人が読みやすい形でまとめたレポートです。

AI入力Markdown

AIに渡しやすい入力整形版です。プレビュー画面で全文確認できます。

推奨 ZIP 作成方法

アップロード時の解凍エラーや構成判定エラーを減らすため、次の形を推奨します。

構成

ZIP を開いた最上位で pom.xml が確認できる単一モジュール構成を推奨します。

除外を推奨するもの
  • .git や .github などの管理ファイル
  • target や build などのビルド成果物
  • 解析に不要な大容量データ、画像、Docker 関連ファイル
エラー時の再試行

解凍に失敗した場合は、標準 ZIP 形式で再圧縮し、不要物を減らしたうえで再アップロードしてください。

現時点で非対応

  • Gradle プロジェクト
  • マルチモジュール構成
  • Spring Framework 単体構成
  • Kotlin 主体プロジェクト
  • マルチリポジトリ横断分析
